# AOF v1.7 Release Checklist

`v1.7` を release-ready と判定する前に確認する checklist。
この文書は [docs/v1.7-release-definition.md](./v1.7-release-definition.md) の gate を、repo 内の evidence に落としたものとする。

## How To Use This Checklist

1. gate ごとに必要 evidence を確認する
2. `v1.7` では orchestrator role, topology, ownership, and schema compatibility を正本とする
3. missing evidence がある場合は `v1.7 ready` と言わない

## Blocking Conditions

次のどれかが残る場合、`v1.7 ready` とは言わない。

1. AI Orchestrator の責務境界が曖昧
2. topology と usage level が混同される
3. parallel execution と independence claim が混同される
4. state ownership が曖昧
5. schema extension が互換性を壊す
