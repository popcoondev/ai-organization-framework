# AOF v3.6.0 Release Notes

- target version: `v3.6.0`
- current package version: `3.6.0`
- status: signed off
- release note revision commit SHA: pending

## Summary

`v3.6.0` is the release that turns AOF visibility into a bounded Mission Control surface with benchmarked runtime stage transitions.

`v3.5.0` made command routing cheaper.  
Running the runtime after that showed the next problem:

- operators could find commands more easily
- but still had to reconstruct mission state from multiple outputs
- the truth existed in artifacts, not yet in one unified operator surface

`v3.6.0` addresses that by making Mission Control a runtime-derived visibility layer rather than a decorative dashboard.

## Highlights

### 1. Mission Overview Becomes First-Class

Mission Control now composes:

- current mission
- current release
- current operating goal
- current next value slice
- current runtime stage

from canonical runtime state.

### 2. Artifact Lineage Is On The Main Operator Path

Mission Control exposes the discovery / need-validation / planning chain as one bounded causal graph so the operator can see why the current state exists.

### 3. Blockers And Next Action Are Unified

Mission Control does not stop at status.  
It also tells the operator:

- what is blocked
- what should happen next

from the same surface.

### 4. Stage Transition Is Benchmarked

`mission-control-benchmark` proves that Mission Control can advance through a real runtime chain:

- `visibility-baseline`
- `discovery-handoff`
- `planning-ready`
- `implementation-ready`

This matters because `v3.6.0` is not just a view layer.  
It is a truthfulness claim about runtime-derived visibility.

## Boundary

`v3.6.0` does not claim:

- pixel-office presentation
- speculative live role-state or council-state views
- new authority boundaries
- broader autonomy

It claims a narrower thing:

**the runtime can expose one truthful Mission Control surface and prove that it tracks the governed artifact chain.**
