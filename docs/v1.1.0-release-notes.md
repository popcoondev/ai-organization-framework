## Summary

- `v1.1.0` is the first AOF release that makes process telemetry and outcome writeback part of the core local runtime surface.
- The release keeps the `v1.0.0` core lifecycle intact while adding machine-readable session transitions, routing history, reopen tracking, and `outcome-report`.
- `v1.1.0` also fixes the runtime usage boundary in docs and runtime behavior by making same-session mutation explicitly serial-only.

## Included In v1.1.0

- `stage_transitions[]` in the session schema and runtime persistence
- `routing_mode_history[]`
- `reopen_count`
- `outcome_reports[]`
- `outcome-report` CLI
- explicit `framing-only` vs `runtime-on` guidance
- same-session mutation lock for mutating runtime commands
- committed example session showing transition telemetry
- committed example session showing outcome writeback

## What v1.1.0 Does Not Claim

- advanced analytics dashboards
- cross-session comparison rollups
- safe multi-writer concurrency
- hosted metrics backend
- richer policy DSL formalization

## Known Limitations

- measurement is collected first; analysis remains mostly future work
- same-session mutation is guarded locally, but stale `.lock` files may require manual cleanup after abnormal process termination
- tag-time CI confirmation should still be checked against the current release candidate commit
