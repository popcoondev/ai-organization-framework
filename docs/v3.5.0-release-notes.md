# AOF v3.5.0 Release Notes

- target version: `v3.5.0`
- current package version: `3.5.0`
- status: signed off
- release note revision commit SHA: `tracked by tag v3.5.0`

## Summary

`v3.5.0` is the release that turns the AOF CLI surface into a governed command-routing layer instead of a large reference the operator or AI must reread every time.

`v3.4.0` made the active release baseline truthful.  
Running that runtime in practice then exposed a new friction:

- the CLI surface had become broad enough that routing cost mattered
- the problem was not missing commands
- the problem was that command discovery still depended on rereading too much context

`v3.5.0` addresses that with a canonical command catalog, a first-class command registry artifact, routing summary integration in the recognition packet, and a dedicated routing audit.

## Highlights

### 1. CLI Taxonomy Is Canonical

The runtime now classifies commands under five categories:

- `read`
- `verify`
- `write`
- `execute`
- `observe`

This taxonomy is no longer implicit operator knowledge. It is part of the runtime.

### 2. Command Registry Is First-Class

The runtime can now materialize `.aof/command-registry.json`.

That registry records, for every supported command:

- command name
- category
- purpose
- operator path
- top command status
- minimal input / output hints

### 3. Recognition Packet Can Route From Minimal Context

The installer surface now wires command routing into the recognition packet:

- `.aof/project-bootstrap.json` includes `command_registry_ref`
- `project-orientation.json` includes `command_routing_summary`
- top commands and runtime flow are available without loading the full CLI reference

### 4. Operator Surfaces Expose Routing Directly

New surfaces:

- `command-register`
- `command-routing-audit`

Updated surfaces:

- `organization-status`
- `organization-verify`
- `init`
- `upgrade`

This release makes command routing inspectable and auditable rather than informal.

## Boundary

`v3.5.0` does not claim:

- broader multi-agent autonomy
- Discovery authority expansion
- Need Validation relaxation
- backend-specific orchestration upgrades

It claims a narrower thing:

**the runtime can expose, route, and audit its command surface with materially lower context cost.**
