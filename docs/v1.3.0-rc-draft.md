## Summary

- `v1.3` is currently defined as an operating-model release, not a runtime-automation expansion
- the release target is to make discovery-driven team assembly, operating goal sizing, autonomy boundaries, and mission control explicit
- current repo state now has the release definition, dedicated operating-model docs, and one concise evolving-task validation artifact

## Scope Of This Release

### Included

- Discovery as acceleration
- team assembly from sizing
- operating goal sizing
- autonomy budget
- escalation boundary
- mission control and drift loop

### Explicitly Not Core In v1.3

- interpretation lens expansion
- cross-domain minimal-model validation as a gate by itself
- advanced verification/dashboard growth
- hosted orchestration
