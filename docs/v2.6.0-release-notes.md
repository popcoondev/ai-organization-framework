# AOF v2.6.0 Release Notes

- version: `v2.6.0`
- status: release-ready
- release note revision commit SHA: `pending release commit (worktree based on 0fc515f)`

## Summary

`v2.6.0` is the release that makes the Human Visibility Layer runtime-backed instead of fixture-backed.

This release adds first-class runtime export for:

- `status_card`
- `timeline_feed`
- `flow_snapshot`

`v2.5.0` made governed allocation inspectable as artifacts.  
`v2.6.0` makes those runtime artifacts projectable into an operator-facing visibility packet.

## Highlights

### 1. Visibility Export Becomes Canonical

`v2.6.0` adds:

- `aof visibility-export`

This command exports:

- `status-card.json`
- `timeline-feed.json`
- `flow-snapshot.json`

from live `.aof` state into a viewer-ready packet directory.

### 2. Visibility Views Become Schema-Backed

`v2.6.0` adds explicit schemas for:

- status card view
- timeline feed view
- flow snapshot view

This keeps the operator-facing visibility surface contract stable enough to validate and reuse.

### 3. Visibility Track Becomes First-Class In Roadmap Status

`roadmap-status` now exposes `v2.6` as a dedicated track so visibility projection work is visible before the repo crosses into `v3.0` runtime proof work.

## Boundary

`v2.6.0` is a visibility projection release.

It still stops before:

- hosted dashboard infrastructure
- backend-neutral end-to-end runtime loop proof
- autonomous orchestration
- `v3.0` execution runtime claim
