# AOF v5.0 Release Definition

`v5.0.0` の主題は、
**Skillful Actor Runtime によって、AOF actor を generic role prompt ではなく、skill / capability / resource / policy / review evidence に基づいて稼働させること**
である。

## Position

`v4.0.0` と `v4.0.1` で AOF は runtime truth を Human Recognition Interface として見えるようにした。

しかし、見える組織が本当に強い組織であるためには、次が必要である。

- 誰を actor として動かすのかを skill evidence で説明できる
- actor が必要とする capability / resource / policy boundary を execution 前に見える化できる
- weak assignment や missing evidence が成功扱いされない
- actor state が Human Recognition Interface に canonical artifact から投影される
- fake success を negative benchmark で落とせる

したがって `v5.0.0` は autonomy expansion ではない。  
`v5.0.0` は **actor が仕事を始める前に、その actor がなぜその仕事をするべきかを runtime が証拠化する release** である。

## v5.0 Goals

1. actor skill packet contract が存在する
2. actor skill packet writer が runtime command として存在する
3. actor assignment が capability-fit evidence で judge できる
4. execution gate が resource claim と policy evaluation を見る
5. missing skill evidence / weak assignment / stale release / policy bypass が benchmark で failure になる
6. Skillful Actor state が Human Recognition Interface に artifact-derived に投影される
7. release closure 自体が runtime-backed task と verification evidence によって閉じられる

## Out Of Scope

- broad autonomous execution claims
- backend-specific agent process management
- decorative character UI not backed by runtime artifacts
- replacing council authority with actor packet authority
- changing Discovery or Need Validation boundaries
- claiming that skillfulness is fully measured by a single score

## Release Gates

### Gate 1: Actor Skill Packet Contract

- actor skill packet schema exists
- contract doc explains assignment, skill, capability, resource, policy, output, review, blocker, and HRI projection fields
- packet semantics distinguish governed assignment from generic prompting

### Gate 2: Governed Assignment

- actor assignment evaluation can select, degrade, or block assignment
- capability-fit evidence is explicit
- missing or weak evidence does not silently pass

### Gate 3: Resource And Policy Execution Gate

- execution gate reads assignment evaluation
- resource claim evidence is required when resource refs exist
- policy evaluation evidence is required when policy refs exist
- gate can produce allowed, degraded, blocked, or requires-council-review states

### Gate 4: Negative Benchmark Coverage

- missing skill evidence fails
- weak actor assignment fails
- stale release state fails
- policy-bypassed allocation fails

### Gate 5: Human Recognition Projection

- Skillful Actor state is projected into Human Recognition Interface artifacts
- actor, skill, confidence, blocker, council review need, and next action are visible from canonical artifacts
- viewer-facing state does not become a second source of truth

### Gate 6: Release Closure Integrity

- package metadata, bootstrap version, active release manifest, governance release contract, release definition, checklist, and notes all point to `5.0.0`
- `TASK-048` through `TASK-055` are closed
- post-release runtime state has no open shipped-release task
- next frontier remains explicitly deferred to a runtime-backed direction review

## Intended Outcome

After `v5.0.0`, AOF can say:

```text
this actor is assigned to this work
this is the skill being invoked
this is the capability evidence
these are the required resources and policies
this is the execution gate decision
this is how council/review should see it
this is how the operator sees the actor state
and fake skillful success is rejected by benchmark coverage
```
