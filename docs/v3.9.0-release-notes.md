# AOF v3.9.0 Release Notes

- target version: `v3.9.0`
- current package version: `3.9.0`
- status: signed off
- release note revision commit SHA: pending

## Summary

`v3.9.0` is the release that turns the operator path from a compact answer into a bounded answer-to-progress, answer-to-tree, and answer-to-proof surface.

`v3.8.0` solved one real problem:

- the runtime could answer `what / why / blocked / next`

Running that runtime in the viewer exposed the next one:

- the viewer still felt static
- change over time was weak
- branch position inside the larger roadmap was weak
- proof existed in artifacts, but was not presented as a bounded operator path
- viewer startup still depended on a manual export + serve + browser ritual

`v3.9.0` addresses that by adding first-class progress, tree-position, evidence drill-down, and viewer-session orchestration below the operator brief.

## Highlights

### 1. Operator Progress Becomes First-Class

`operator-progress` now explains:

- what changed since the last meaningful checkpoint
- what the current frontier is now
- why that change matters

### 2. Tree Position Becomes First-Class

`tree-position` now explains:

- which shipped release trunk is current
- which frontier branch the runtime is actively working on
- why this branch is the current path

### 3. Evidence Drill-Down Becomes First-Class

`evidence-drill-down` now explains the bounded proof path for:

- the current headline
- the current blocker state
- the current recommended next action

### 4. Viewer Session Becomes One Runtime Path

`visibility-session` now:

- exports the current visibility packet
- starts the viewer session
- can open the browser from the same runtime command

This makes the upgrade visible in real use instead of leaving viewer wiring as a manual side ritual.

### 5. Visibility Export Now Includes Progress / Tree / Proof

`visibility-export` now writes:

- `status-card.json`
- `timeline-feed.json`
- `flow-snapshot.json`
- `mission-control.json`
- `operator-brief.json`
- `operator-progress.json`
- `tree-position.json`
- `evidence-drill-down.json`

### 6. Viewer Surface Shows Meaningful Change

The viewer can now surface:

- `Now`
- `What Changed`
- `Where In The Tree`
- `Evidence Drill-Down`

This is the intended value upgrade over the mostly static v3.8 surface.

## Boundary

`v3.9.0` does not claim:

- decorative observability authority
- analytics-first dashboarding
- plugin-heavy browser orchestration
- a second truth model outside canonical runtime artifacts
- new execution authority

It claims a narrower thing:

**AOF can make its operator progress, branch position, proof path, and viewer session legible from one truthful runtime path.**
