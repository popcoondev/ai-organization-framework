## Summary

`v1.8.0` is planned as the release that gives AOF native ownership of project memory.

Where `v1.7.0` clarified who orchestrates multi-session governance, `v1.8.0` is intended to explain:

- how AOF keeps its own task ledger
- how goals are projected inside `.aof/`
- how Alignment Pulse triages tasks
- how Archivist manages task and thread archive
- how AOF audits itself for missing coverage

## Planned Scope

- task management model
- `.aof/` directory structure
- `Task Triage` in alignment pulse
- Archivist scope expansion
- framework self-audit
- schemas for tasks and goal projection

## Planned Non-Goals

- GitHub Issues sync as source of truth
- multi-session viewer expansion
- task UI/dashboard
- full orchestration daemon
