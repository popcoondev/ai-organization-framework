# AOF v1.6 Release Checklist

`v1.6` を release-ready と判定する前に確認する checklist。  
この文書は [docs/v1.6-release-definition.md](./v1.6-release-definition.md) の gate を、repo 内の evidence と experience-validation docs に落としたものとする。

## How To Use This Checklist

1. gate ごとに必要 evidence を確認する
2. `v1.6` では experience-validation と proof taxonomy を正本とする
3. missing evidence がある場合は `v1.6 ready` と言わない

## Blocking Conditions

次のどれかが残る場合、`v1.6 ready` とは言わない。

1. experience-validation が phase ではなく単なる advice に留まる
2. Experience Steward の責務が曖昧
3. viewer fidelity と editor fidelity が未分離
4. `expectation-mismatch` が first-class signal として定義されていない
5. proof category taxonomy が無く、spec completeness と product-feel truth が混同される
