# AOF v1.2 Release Definition

AI Organization Framework を `v1.2` として更新・リリースするための基準を定義する。

この文書の目的は 3 つある。

1. `v1.1` の measurement foundation の次に、何を concept layer として固定するかを明示する
2. `Need` 前提のモデルから、`Need` を発見するモデルへの拡張境界を固定する
3. `v1.2` に入れる概念と、以後の heavier formalization を切り分ける

## Position

`v1.2` は analytics 完成版でも hosted product でもない。  
`v1.1` で測定基盤と runtime usage policy を固定したうえで、

- request がまだ曖昧な段階をどう扱うか
- 最初の組織サイズをどう決めるか
- どの方法論の目線で request を解釈するか

を concept layer として固定する minor release である。

言い換えると、`v1.0` が最初の公開版、`v1.1` が観測基盤版なら、`v1.2` は
**曖昧な request を受けて、適切な初期組織で始めるための版** である。

## v1.2 Goal

`v1.2` の主目標は次である。

- `Need` がまだ存在しない request を AOF が受けられるようにする
- `Discovery Layer` と `Knowledge State` を最小概念として固定する
- `scope sizing -> initial organization selection` を lightweight rule として固定する
- Design Thinking / JTBD / User Story / PMBOK / AIDLC を `interpretation lens` として位置づける

要するに、`v1.2` は feature expansion の版ではなく、
**AOF が曖昧な request をどう解釈し、どの大きさの組織で始めるかを固定する版** である。

## In Scope For v1.2

### Discovery Layer

- `Unknown -> Question -> Answer -> Insight -> Need` を AOF の concept layer に追加する
- `Need / Intent / Context` の前に discovery がありうることを明文化する
- clarification を「不足補完」だけでなく「Need discovery」の一部として位置づける

### Knowledge State

- minimum state:
  - `known`
  - `unknown`
  - `assumptions`
  - `confidence`
- 何を知っていて何を知らないかを判断可能にする
- 質問優先順位と clarification stop condition の根拠を与える

### Organization Pattern (Lightweight)

- heavy taxonomy ではなく lightweight initial sizing に留める
- minimum size classes:
  - `small`
  - `medium`
  - `large`
  - `extended`
- 各 class に対して default initial organization を与える

### Scope Sizing And Initial Organization Selection

- initial sizing を `Facilitator` の一次責務として定義する
- minimum sizing axes:
  - `scope breadth`
  - `lifecycle span`
  - `stakeholder diversity`
  - `risk / irreversibility`
  - `outcome horizon`
- 上記から初期組織サイズを提案する rule を持つ

### Interpretation Lenses

- AOF core とは別に、次を interpretation lens として扱う
  - Design Thinking
  - JTBD
  - User Story
  - PMBOK
  - AIDLC
- minimum mapping を固定する
  - `Discovery`: Design Thinking + JTBD
  - `Intent / delivery shaping`: User Story
  - `organization sizing / governance`: PMBOK
  - `AI/software workflow mapping`: AIDLC

### Documentation And Validation

- README か companion docs で、AOF が「AI をどう動かすか」より「曖昧な request をどう判断可能にするか」の規格であることを明確化する
- minimal-model validation を、少なくとも複数ドメインで説明可能な形に保つ

## Out Of Scope For v1.2

### Heavy Formalization

- full Policy DSL
- large actor taxonomy
- domain-specific workflow catalogs
- enterprise / regulated pattern の詳細規格化

### Advanced Runtime Expansion

- multi-writer concurrency
- branch/merge session semantics
- advanced telemetry analytics
- hosted discovery workspace

### Productization

- UI product surface
- backend metrics service
- team administration model

## Non-Goals

`v1.2` の non-goal は次である。

1. discovery を完璧な user research framework にしない
2. initial organization sizing を heavy process asset にしない
3. methods library を AOF core modelそのものに混ぜない
4. actor 数を増やすこと自体を成果にしない
5. Policy DSL を先に固定しない

## Deferred To v1.3+

### Discovery Expansion

- richer discovery artifacts
- research session archives
- domain-specific discovery packs

### Organization Expansion

- enterprise / regulated pattern formalization
- multi-council coordination
- stronger delegation and exception handling

### Measurement Expansion

- cross-session comparison rollups
- discovery-vs-delivery correlation reports
- sizing accuracy review based on actual outcomes

## Release Gates

### Gate 1: v1.1 Integrity Is Preserved

- `v1.1.0` measurement foundation still works
- `v1.1.0` release surface is not regressed
- current templates and release docs remain coherent

### Gate 2: Discovery Layer Is Explicit

- docs clearly define the pre-Need discovery flow
- AOF no longer assumes every request already has a valid `Need`
- clarification/discovery relationship is explained coherently

### Gate 3: Knowledge State Is Minimal And Useful

- minimum knowledge state fields are named and justified
- docs explain how they reduce redundant questioning and unsupported assumptions
- the model is still lightweight enough for human/front-AI use

### Gate 4: Initial Organization Sizing Is Real

- a lightweight size class model exists
- sizing axes are explicit
- default initial organization selection is described

### Gate 5: Interpretation Lenses Are Mapped

- the five lenses are named
- their role is clearly outside the AOF core model but inside AOF operation
- the mapping from lens to AOF phase is readable and non-contradictory

## Release Evidence

`v1.2` readiness を主張するときは、最低限次を示す。

1. current commit SHA
2. docs showing Discovery Layer and Knowledge State
3. docs showing lightweight organization sizing
4. docs showing interpretation lens mapping
5. at least one concise validation artifact proving the model still spans multiple domains

## Current Intended Shape

現時点での `v1.2` の intended shape は次である。

- `collect first` の次は `interpret first`
- discovery と sizing を core candidate として固定する
- pattern は軽く始め、taxonomy は急がない
- AOF core model と interpretation lens を混ぜない

## Relationship To v1.1

`v1.1` は process/outcome を測れるようにした。  
`v1.2` は、その前段として「そもそも何を解くのか」「どの大きさの組織で始めるのか」を
より明示的に扱う。

したがって、`v1.2` の完了条件は analytics を増やすことではなく、
**AOF が discovery-to-delivery の入口を持つことを concept level で固定すること** にある。
