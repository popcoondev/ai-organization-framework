# AOF v1.5 Release Definition

AI Organization Framework を `v1.5` として更新・リリースするための基準を定義する。

`v1.5` の主題は、これまで暗黙に置かれていた **信頼前提と運用前提** を明文化することにある。

## Position

`v1.5` は runtime 強化版でも viewer 拡張版でもない。  
`v1.4` で Human Visibility Layer を固定したうえで、

- どの前提なら Fast Track を信用してよいか
- どの execution pattern なら Council of Three の独立性をどこまで主張できるか
- Clarification / Discovery の出力品質をどこで止めるべきか
- 人間レビューがどの頻度・深さで必要か
- 構造化出力や context 読み取りの限界をどう扱うか

を **trust assumptions** として固定する版である。

言い換えると、`v1.4` が「runtime を人間に見えるようにした版」なら、`v1.5` は  
**フレームワークを信頼して使うための前提条件を明示した版** である。

## v1.5 Goal

`v1.5` の主目標は次である。

- routing judgment が model capability に依存することを明示する
- Council execution pattern ごとの独立性強度を定義する
- Clarification / Discovery の最低品質基準と stop condition を定義する
- usage level ごとの human review cadence を定義する
- structured output fallback と `unknown` escalation の扱いを仕様化する
- context 完全性の最低確認項目を運用 guidance に入れる

## In Scope For v1.5

### Routing Trust Assumptions

- Fast Track 判定に必要な model capability assumption を定義する
- model capability 不明時の default routing policy を定義する
- `blast radius / reversibility / ambiguity / safety impact` を読む主体の限界を明記する

### Council Independence And Execution Patterns

- `single-instance / hybrid / multi-agent` の execution pattern を定義する
- 各 pattern でどの程度の disagreement assurance を主張できるかを明示する
- high-risk / irreversible decision で要求される independence level を guidance として置く

### Clarification / Discovery Quality Floor

- weak model が見落としやすい unknown や gap の扱いを定義する
- Discovery / Clarification の minimum quality floor を定義する
- unresolved critical unknown が残る場合の stop / escalate condition を定義する
- `confidence` を self-assessed であり calibration 済みではないと明記する

### Human Review Cadence

- `framing only / partial runtime / runtime mandatory` ごとの minimum review cadence を定義する
- review depth と gate frequency の最小要件を置く
- escalation を「モデル不能時」だけでなく「人間レビュー必須時」にも結びつける

### Structured Output Reliability

- model adapter が structured output を期待するが保証しないことを明記する
- natural-language fallback の存在と限界を明記する
- critical stage で `unknown` が返った場合の escalation rule を定義する

### Context Completeness

- brownfield / orientation で最低限確認すべき context checklist を定義する
- human owner が明示入力すべき制約カテゴリを guidance として置く

## Out Of Scope For v1.5

- stronger model routing automation
- full multi-model orchestration backend
- probabilistic calibration engine
- automated context extraction from arbitrary legacy systems
- rich reviewer staffing libraries
- hosted governance assurance service

## Non-Goals

1. AI の判断を「完全に信頼できる」と主張しない
2. single-instance execution に本当の独立性があると装わない
3. `confidence` を calibrated score と誤認させない
4. review cadence を product-specific workflow まで固定しすぎない
5. fallback parser の存在を correctness guarantee と誤認させない
6. 運用 guidance を重い enterprise process にしない

## Release Gates

### Gate 1: v1.4 Integrity Is Preserved

- visibility vocabulary and output formats remain coherent
- `static AOF / runtime-used AOF` distinction remains valid
- trust-assumption docs do not contradict `v1.4` visibility semantics

### Gate 2: Routing Capability Assumptions Are Explicit

- docs define capability dependence of routing
- docs define default-to-Deep-Path guidance for unknown/weak models
- docs explain why routing trust is conditional

### Gate 3: Council Independence Limits Are Explicit

- docs define execution patterns and their independence strength
- docs state that single-instance role switching does not guarantee independent disagreement
- docs connect high-risk decisions to stronger execution patterns

### Gate 4: Clarification / Discovery Quality Limits Are Explicit

- docs define minimum quality expectations
- docs define critical-unknown stop/escalate conditions
- docs mark `confidence` as self-assessed rather than calibrated

### Gate 5: Human Review Cadence Is Explicit

- docs define minimum review cadence by usage level
- docs connect cadence to escalation timing
- docs show that “human as backstop” is operationally defined, not merely stated

### Gate 6: Output / Context Reliability Limits Are Explicit

- docs define structured-output fallback limits
- docs define `unknown` handling at critical stages
- docs define minimum context completeness checks

## Deferred To v1.6+

- quantitative model-capability certification
- automated routing confidence estimation
- richer execution-pattern selection tooling
- stronger human-review workload optimization
- runtime-enforced context completeness checks

## Current Intended Shape

`v1.5` は、AOF をより万能にするために「何でも自動化する」版ではない。  
重点は、利用者が

- どこまで AI 判断を信頼してよいか
- どこで人間が入るべきか
- どの execution pattern を選ぶべきか
- どの quality floor を満たさなければ止まるべきか

を事前に理解できるようにすることにある。

したがって `v1.5` は、AOF の trust boundary と operating assumptions を明示する release とする。
