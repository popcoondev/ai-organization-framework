# AOF v3.7.0 Release Notes

- target version: `v3.7.0`
- current package version: `3.7.0`
- status: signed off
- release note revision commit SHA: pending

## Summary

`v3.7.0` is the release that turns post-release drift into a bounded runtime diagnosis problem instead of an operator reconstruction problem.

`v3.6.0` made mission state more legible.  
Running the released runtime after that exposed the next issue:

- the active release could be correct while roadmap guidance stayed stale
- shipped implementation work could linger on the main operator path
- Mission Control could still recommend old release work
- the operator still had to explain why those surfaces disagreed

`v3.7.0` addresses that by adding a runtime-native situation assessment layer and wiring roadmap and Mission Control through it.

## Highlights

### 1. Situation Assessment Becomes First-Class

`situation-assess` now emits a bounded diagnosis packet that names:

- the active release
- the current runtime stage
- the primary frontier task
- current truth conflicts
- the recommended next action

### 2. Post-Release Drift Stops Looking Like Current Work

The runtime can now distinguish:

- shipped implementation work
- current frontier work
- stale alignment state

This matters because AOF should not keep recommending already-released work.

### 3. Roadmap Guidance Follows Live Truth

`roadmap-status` now derives current alignment from:

- the active release manifest
- the current goal surface
- the current live frontier task

instead of letting historical decomposition dominate present guidance.

### 4. Mission Control Uses Diagnosis Rather Than Viewer-Only State

Mission Control still exists, but it now follows runtime judgment rather than stale release residue.  
That keeps visibility additive while making the operator path more truthful.

## Boundary

`v3.7.0` does not claim:

- new execution authority
- broader autonomy
- decorative viewer expansion
- speculative role-state or council-state liveness

It claims a narrower thing:

**the runtime can diagnose post-release truth conflicts and keep its main operator surfaces aligned with the real current frontier.**
