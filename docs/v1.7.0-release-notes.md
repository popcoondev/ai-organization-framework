## Summary

`v1.7.0` is planned as the release that makes **AI Orchestrator** a first-class part of AOF.

Where `v1.6.0` clarified value alignment and experience validation, `v1.7.0` is intended to explain:

- who operates the loop between Human and worker threads
- how execution topology differs from usage level
- why parallel execution does not automatically imply independent judgment
- how state ownership and aggregation should work
- which schema additions are needed before orchestration becomes first-class

## Planned Scope

- AI Orchestrator role
- execution topology
- independence strength
- parallel execution model
- state ownership and aggregation failure handling
- schema extensions for sessions and decision records

## Planned Non-Goals

- multi-session visibility implementation
- orchestrator daemon implementation
- adaptive thread budgets
- L4 cross-orchestrator coordination

## Non-Core But Preserved

`v1.7` の主題は orchestrator governance であり、viewer implementation ではない。  
ただし `v1.4` の visibility layer、`v1.5` の trust assumptions、`v1.6` の value alignment / experience validation は preserved foundation として維持する。

## Current Reading

- companion docs and schema extensions are assembled in the repo
- runtime core rewrite remains intentionally out of scope
- `visibility-serve` multi-session expansion remains intentionally out of scope
