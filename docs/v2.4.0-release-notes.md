# AOF v2.4.0 Release Notes

- version: `v2.4.0`
- status: draft
- release note revision commit SHA: `TBD (uncommitted release candidate worktree)`

## Summary

`v2.4.0` is the release that turns execution and judgment handoff into explicit AOF artifacts.

This release adds first-class packet surfaces for:

- role results
- team outputs
- council reviews
- execution lineage

`v2.3.0` made the organization inspectable.  
`v2.4.0` makes the work moving through that organization inspectable.

## Highlights

### 1. Role Execution Results Become Canonical

`v2.4.0` adds:

- `aof role-result-record`

This command records a role-level execution result with:

- role / stage / session identity
- completion state
- recommendation and rationale
- signals
- artifact refs
- decision-required state

This is the first explicit packet boundary between role work and team aggregation.

### 2. Team Aggregation Becomes Canonical

`v2.4.0` adds:

- `aof team-output-record`

This command records:

- expected roles
- received roles
- missing roles
- aggregate state
- blocking signals
- joined role result refs
- recommended next step

This lets AOF explain whether a team packet is ready for review, waiting on a role, or degraded.

### 3. Council Judgment Becomes Canonical

`v2.4.0` adds:

- `aof council-review-packet`

This command records:

- review status
- decision summary
- rationale
- recommendation
- team / role evidence refs
- follow-up task ids
- escalation requirement

This gives AOF a structured judgment artifact between team execution and next-task governance.

### 4. Execution Lineage Is Now Queryable

`v2.4.0` adds:

- `aof execution-lineage`

This command aggregates execution artifacts and writes an active lineage snapshot that summarizes:

- role result / team output / council review counts
- stages observed
- latest stage
- blocking signal count
- recommended next step

This is the first runtime surface that shows the actual execution chain without relying on free-form notes.

## Install / Upgrade

Current canonical acquisition path:

```bash
git clone --branch v2.4.0 https://github.com/popcoondev/ai-organization-framework.git ~/.local/share/aof/v2.4.0
cd ~/.local/share/aof/v2.4.0
npm install
npm link
```

Then initialize another repository with:

```bash
aof init --topology managed-project
```

Or migrate an existing bootstrap shape with:

```bash
aof upgrade
```

## Boundary

`v2.4.0` is the release where AOF formalizes execution packet contracts.

It still stops before:

- governed allocation
- policy enforcement runtime
- backend-specific orchestration automation
- v3.0 backend-neutral loop proof

The next bridge release, `v2.5.0`, is where allocation planning, resource claims, and policy evaluation should begin.
