## Summary

`v1.10.0` is planned as the release that makes the cadence runtime human-visible through the existing AOF visibility contract, while also formalizing a topology-aware GitHub operations rule.

Where `v1.9.0` made cadence runtime execution and scheduler binding real, `v1.10.0` is intended to explain and demonstrate:

- how cadence timing / dispatch / scheduler profile can travel through `status_card`
- how `visibility-serve` can render cadence summary without new top-level view families
- how self-hosting GitHub Actions binding remains a topology-specific choice
- how managed projects should avoid direct cadence writes to product `main`

## Planned Scope

- cadence summary fields on the human visibility contract
- cadence summary rendering in `visibility-serve`
- topology-aware GitHub operations and branch strategy docs
- hardened self-hosting GitHub cadence workflow semantics

## Planned Non-Goals

- live `.aof` -> visibility producer automation
- specialized cadence dashboard panes
- full managed-project bootstrap templates
- cross-repo scheduler federation

## Current Reading

- cadence runtime surfaces remain intact from `v1.9.0`
- cadence summary is now contract-compatible and viewer-readable
- self-hosting vs managed-project write-target policy is now explicit
- next gap moves to automatic visibility projection from live `.aof` state
