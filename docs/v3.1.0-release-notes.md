## Summary

`v3.1.0` is the release that turns the self-hosting runtime loop from a single proof into a benchmark-grade release surface.

`v3.0.x` proved that one backend-neutral loop could run end to end on the self-hosting repo.  
`v3.1.0` makes that proof more trustworthy by adding:

- review-validity benchmark families
- runtime-discipline failure families
- bounded-manual-review audit replay artifacts
- explicit runtime stage-to-benchmark honesty surfaces

This release does not claim solved autonomy. It claims stronger, replayable evidence for where the runtime is trustworthy and where it is still partial.

## Highlights

### 1. Review Validity Is Normalized As A Release Artifact

`v3.1.0` closes `TASK-035` with:

- `.aof/artifacts/benchmarks/review-validity-family-register-2026-06-16.json`

This register normalizes committed coverage across:

- weak-artifact rejection
- human-low / ai-high disconfirmation
- before/after quality delta
- multi-loop improvement trace

The result is that weak review behavior is no longer only discussed in prose. It is recorded as benchmark evidence with reusable artifact families.

### 2. Runtime Discipline Gains A Low-Cost Audit Shortcut

`v3.1.0` closes `TASK-036` with:

- `.aof/artifacts/benchmarks/runtime-discipline-family-register-2026-06-16.json`
- `.aof/artifacts/benchmarks/runtime-discipline-runs/RDB-20260615T161740Z-audit-shortcut.json`

The runtime-discipline runner now records:

- prose-only orchestration rejection
- three replayable broken-chain families
- machine-readable audit note
- audit packet
- reconstruction map
- audit index
- audit gate
- audit shortcut

This means a human can recheck the latest positive path from one compressed packet plus canonical refs, while the same runner still rejects runtime bypass and missing-chain cases.

### 3. Runtime Coverage Honesty No Longer Depends On One Family

`v3.1.0` keeps the self-hosting proof green while broadening honesty surfaces through:

- `.aof/artifacts/runtime-loop-proofs/proof-family-register-2026-06-16.json`
- `.aof/artifacts/benchmarks/framing-benchmark-family-register-2026-06-16.json`
- `.aof/artifacts/benchmarks/allocation-benchmark-family-register-2026-06-16.json`
- `.aof/artifacts/benchmarks/runtime-stage-benchmark-map-2026-06-16.json`

The release claim is therefore no longer "one happy-path loop exists".  
The claim is "one loop exists, its failure families are partially replayable, and the remaining weak stages are called out explicitly."

## Boundary

`v3.1.0` is a benchmark and coverage-expansion release.

It still stops before:

- always-on autonomous scheduler claims
- human-review removal claims
- fully benchmark-complete diagnosis or outcome coverage
- broad provider-family production parity claims
- generalized workforce-management autonomy

## Operational Note

The repo continues to treat Node 22 LTS as the CI-validated runtime lane.

Local runs on Node 25 can still emit the existing warning that the lane is outside the validated support boundary.  
`v3.1.0` does not change that boundary. It keeps the warning explicit while broadening benchmark evidence on the supported contract surface.

## Honest Weaknesses

The release still surfaces real follow-up signals:

- runtime coverage remains partial for diagnosis, learning, and outcome depth
- smoke still reports verify drift follow-up recommendations such as `investigate-lineage-drift` and `human-review-recommended`
- framing and allocation still need stronger negative verdict families in later work

Those weaknesses remain part of the release story, not something hidden behind a green exit code.
