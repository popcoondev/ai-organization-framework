# AOF v3.3 Release Definition

`v3.3.0` の主題は、AOF を
`Discovery evidence -> Need Validation -> project creation`
の系として release-grade に接続することである。

## Position

`v3.2.0` は Need Validation を mandatory pre-project gate にした。  
ただし upstream にはまだ gap がある。

- discovery evidence は schema-backed だが research-track 扱いが強い
- discovery judgment は release boundary に組み込まれていない
- Need Validation に入る前の evidence assembly はまだ部分的に ad hoc である

`v3.3.0` でやるのは full Discovery autonomy ではない。  
やるのは、Discovery を

- idea expansion layer

としてではなく、

- evidence-bearing pre-Need-Validation layer

として operationalize することである。

## v3.3 Goals

1. Discovery evidence の最小必須 artifact set を next-release contract として固定する
2. Discovery judgment を continue / pivot / handoff / stop の release-grade decision surface にする
3. Discovery-to-Need-Validation handoff を auditable input contract として扱えるようにする
4. Need Validation の upstream evidence quality を benchmarkable にする
5. `v3.2` の Need Validation gate と backend-neutral boundary を後退させない

## In Scope

- `docs/v3.3-release-definition.md`
- `docs/vnext-roadmap.md`
- `docs/v3-discovery-artifact-definition.md`
- `docs/v3-discovery-layer-proposal.md`
- discovery artifact schemas and commands already present in `v3.2`
- discovery-to-need-validation planning evidence on the self-hosting repo
- benchmark or verification surfaces needed to judge discovery evidence completeness before Need Validation

## Out Of Scope

- Discovery だけで project creation を許可すること
- full Discovery Council productization
- autonomous market research execution
- backend-specific multi-agent launcher claims
- human review removal claims

## Release Gates

### Gate 1: Discovery Evidence Contract Is Fixed

- `discovery-question-set-record` is part of the next-release operating path
- `assumption-map-record` is part of the next-release operating path
- `anomaly-log-record` is part of the next-release operating path
- `breakthrough-pattern-record` remains optional but recognized as a reusable strengthening input
- docs clearly state which discovery artifacts are mandatory, optional, and provisional

### Gate 2: Discovery Judgment Is Operationalized

- `discovery-judgment-packet` is treated as a first-class judgment artifact
- `continue-exploration`, `pivot`, `synthesize-handoff`, and `stop` are explicitly bounded
- handoff promotion requires discovery judgment evidence rather than prose-only recommendation

### Gate 3: Handoff Into Need Validation Is Auditable

- `discovery-handoff-record` is runnable through CLI without parser/runtime defects
- handoff packet clearly states selected need, rejected alternatives, risks, and delivery validation requirements
- Need Validation docs and operations docs agree that discovery strengthens input but does not replace the gate

### Gate 4: Upstream Evidence Quality Is Verifiable

- there is a repeatable verification path for discovery-to-need-validation completeness
- weak or missing discovery evidence can be represented as a blocking condition before stronger project creation claims are made
- the self-hosting repo contains at least one committed planning-grade example chain

### Gate 5: Boundary Remains Honest

- `v3.3.0` does not claim solved business-discovery autonomy
- `v3.3.0` does not claim direct project creation from Discovery alone
- `v3.3.0` continues to treat Need Validation as mandatory pre-project authority
- `v3.3.0` continues to avoid backend lock-in and premature autonomy claims

## Verification Snapshot

`v3.3.0` direction readiness should be checked at least by:

- runtime session evidence showing `Discovery -> Need Validation -> planning`
- `node ./src/cli.js organization-verify --project .`
- `node ./src/cli.js discovery-handoff-benchmark --project .`
- targeted regression coverage for discovery handoff CLI/runtime behavior
- `npm run smoke`
- self-review that checks consistency between discovery docs, need-validation docs, and roadmap language

## Intended Outcome

After `v3.3.0`, AOF operator は次を coherent pre-project contract として扱える。

```text
raw question or opportunity
  -> discovery question set
  -> assumption / anomaly / optional breakthrough evidence
  -> discovery judgment
  -> discovery handoff
  -> Need Validation
  -> validated need
  -> project charter
  -> planning
```

この版の価値は、
Need Validation を孤立した gate のままにせず、
**その前にある discovery evidence quality まで governance の対象にする**
点にある。
