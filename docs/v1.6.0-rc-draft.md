## Summary

- `v1.6` is currently defined as an experience-validation release, not a runtime-expansion release
- the release target is to make live experience truth a first-class gate before downstream confidence grows too far
- current repo state now has the release definition, but the companion docs and proof taxonomy evidence are not assembled yet

## Scope Of This Release

### Included

- `experience-validation` as an all-product phase
- `Experience Steward` as a companion role
- viewer fidelity and editor / authoring fidelity split
- `expectation-mismatch` as a first-class signal / runtime state
- proof category taxonomy

### Explicitly Not Core In v1.6

- automated UX scoring
- screenshot-diff automation platform
- hosted visual review gallery
- quantitative experience confidence engine
- domain-specific style packs for every product class
