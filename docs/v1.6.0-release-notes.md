## Summary

`v1.6.0` is planned as the release that makes **live experience truth** and **value alignment** first-class parts of AOF.

Where `v1.5.0` clarified trust assumptions and operating assumptions, `v1.6.0` is intended to explain:

- when experience validation must happen before downstream confidence grows
- how `North Star Goal`, `Current Operating Goal`, and `Next Value Slice` stay connected
- why short repeated confirmation is better than one long intake
- who is responsible for validating intended feel
- why viewer fidelity and editor / authoring fidelity must be separated
- how `expectation-mismatch` should be raised and handled
- why specification proof is not enough without screenshot / interaction / product-feel proof

## Planned Scope

- 3-layer goal model
- `Value Alignment Loop`
- `experience-validation` phase
- `Experience Steward` companion role
- viewer fidelity / editor fidelity split
- `expectation-mismatch` signal
- proof category taxonomy

## Planned Non-Goals

- automated visual truth scoring
- hosted review gallery
- computer-vision-based expectation matching
- quantitative experience confidence engine
- universal taste automation

## Non-Core But Preserved

`v1.6` の主題は value alignment と experience validation であり、viewer expansion ではない。  
ただし `v1.4` までで追加された human visibility surface と `v1.5` の trust-assumption layer は preserved foundation として維持する。

- `status_card`
- `timeline_feed`
- `flow_snapshot`
- `visibility-serve`
- trust assumptions and review cadence guidance
