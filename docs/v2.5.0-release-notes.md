# AOF v2.5.0 Release Notes

- version: `v2.5.0`
- status: draft
- release note revision commit SHA: `TBD (uncommitted release candidate worktree)`

## Summary

`v2.5.0` is the release that makes governed allocation reviewable before execution begins.

This release adds first-class packet surfaces for:

- allocation plans
- policy evaluation reports
- resource claims
- workload-aware allocation metrics

`v2.4.0` made execution inspectable.  
`v2.5.0` makes pre-execution assignment reasoning inspectable.

## Highlights

### 1. Allocation Recommendations Become Canonical

`v2.5.0` adds:

- `aof allocation-plan-record`

This command records:

- subject ref
- target role refs
- candidate resource refs
- recommended allocations
- policy refs
- risk notes

This is the first explicit packet boundary for governed assignment planning.

### 2. Policy Evaluation Becomes Canonical

`v2.5.0` adds:

- `aof policy-evaluation-report`

This command records:

- evaluation scope
- policy refs
- overall outcome
- per-policy results
- recommended actions

This lets AOF explain whether an allocation or execution request is allowed, review-gated, approval-gated, escalated, or denied.

### 3. Resource Claims Become Canonical

`v2.5.0` adds:

- `aof resource-claim-record`

This command records:

- resource ref
- claimant role ref
- claim scope
- claim status
- approval policy refs
- justification
- optional allocation plan and policy evaluation refs

This gives AOF a structured transition from recommendation into explicit reservation / approval state.

### 4. Allocation Review Load Becomes Visible

`v2.5.0` extends:

- `aof metrics-snapshot`

This snapshot now reports:

- allocation plan count
- policy evaluation count
- resource claim count
- approval-required allocation count
- allocation review load
- open resource claim count

This keeps `v2.5.0` honest as a governed allocation release rather than a pure packet-format release.

## Install / Upgrade

Current canonical acquisition path:

```bash
git clone --branch v2.5.0 https://github.com/popcoondev/ai-organization-framework.git ~/.local/share/aof/v2.5.0
cd ~/.local/share/aof/v2.5.0
npm install
npm link
```

Then initialize another repository with:

```bash
aof init --topology managed-project
```

Or migrate an existing bootstrap shape with:

```bash
aof upgrade
```

## Boundary

`v2.5.0` is the release where AOF formalizes governed allocation artifacts.

It still stops before:

- autonomous resource scheduling
- runtime policy enforcement
- backend-neutral end-to-end loop proof
- `v3.0` execution orchestration claim

The next release, `v3.0.0`, is where allocation, execution, review, outcome, and next-step recommendation must connect into one auditable runtime loop.
