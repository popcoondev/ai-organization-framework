# AOF v1.6 Release Definition

AI Organization Framework を `v1.6` として更新・リリースするための基準を定義する。

`v1.6` の主題は、AOF が specification / governance / visibility に強い一方で、**live experience の妥当性** と **ユーザー期待との継続的な整合** を十分に早く検証できないという弱点を補うことにある。

## Position

`v1.6` は viewer 拡張版でも trust-boundary 追加版でもない。  
`v1.5` で trust assumptions を固定したうえで、

- 仕様として整っていても、体験としてズレている状態を早く検出する
- `North Star Goal` と `Next Value Slice` を同時に保ちながら進める
- 同じ core question を短く再確認する value alignment loop を回す
- visual / spatial product だけでなく、あらゆる product に対して live experience validation を要求する
- specification proof と product-feel proof を分ける
- experience mismatch を first-class runtime state として扱う
- experience validation を担保する companion role を置く
- uncertainty が高い案件では runtime default を強める

ことを目的とする版である。

言い換えると、`v1.5` が「どこまで信頼してよいかを明示した版」なら、`v1.6` は  
**その信頼を live experience と value alignment loop によって継続検証する版** である。

## v1.6 Goal

`v1.6` の主目標は次である。

- `North Star Goal / Current Operating Goal / Next Value Slice` の 3 層 goal model を明示する
- `Value Alignment Loop` を cross-cutting loop として定義する
- all-product phase として `experience-validation` を追加する
- `Experience Steward` を AOF companion role として定義する
- viewer fidelity と editor / authoring fidelity を分離して扱う
- `expectation-mismatch` を first-class signal / runtime state として定義する
- screenshot review / interaction proof / live artifact を required proof に加える
- proof category taxonomy を導入して、spec completeness と experience truth を混同しないようにする

## In Scope For v1.6

### Goal Layers And Value Alignment Loop

- `North Star Goal`
- `Current Operating Goal`
- `Next Value Slice`

の 3 層を同時に扱う model を定義する。

- `North Star Goal`
  - 最終完成像
- `Current Operating Goal`
  - 今の checkpoint 群の前進条件
- `Next Value Slice`
  - 次に見せる動く価値、または visible proof

また、短い repeated confirmation を行う `Value Alignment Loop` を定義する。

- 前回と同じ問いを再確認してよい
- recent confirmation window を優先的に参照する
- alignment pulse を long-running work の cadence check として持つ
- expectation mismatch が出たら loop に戻る

### Experience Validation Phase

- `experience-validation` を product-agnostic phase として定義する
- value slice build の後、次の scale-up 前に置く
- minimum required outputs を定義する
  - live preview artifact
  - screenshot set
  - interaction proof
  - expectation alignment note
- exit rule を定義する
  - live artifact が intended experience に sufficiently aligned していなければ downstream confidence を上げない

### Experience Steward Role

- `Experience Steward` を companion role として定義する
- responsibility を定義する
  - expected feel の明文化
  - screenshot / interaction / live artifact の要求
  - `expectation-mismatch` signal の起票
  - specification proof と product-feel proof の切り分け
- `Visionary / Builder / Guardian` を置き換えず、experience truth を補完する立場として置く

### Fidelity Split

- `viewer fidelity` と `editor fidelity` を別 judgment に分ける
- 軽量 viewer 判断が、authoring interaction の妥当性を保証しないことを明示する
- visual / spatial / authoring-heavy products では editor expectation を first-class architecture input とみなす

### Expectation-Mismatch Signal

- `expectation-mismatch` を first-class signal として定義する
- これは bug / blocker / architecture failure と同一視しない
- 役割を定義する
  - logically consistent だが intended experience とズレている状態を記録する
  - reprioritize / reframe / re-architecture の起点にする

### Authoring Expectation Unknowns

- creator tool を含む product では、authoring expectation unknowns を high-impact unknown として扱う
- examples を定義する
  - 2D vs 3D placement
  - direct manipulation vs inspector-only editing
  - fake preview vs real preview
  - timeline vs node graph
- unresolved authoring expectation unknowns は planning blocker になりうると明示する

### Proof Category Taxonomy

- proof category を分ける
  - specification proof
  - governance proof
  - runtime proof
  - screenshot proof
  - interaction proof
  - product-feel proof
- これらを混同しない運用 guidance を置く
- specification completeness が product-feel validity を保証しないことを明示する

### Runtime Default By Uncertainty

- framing-only を完全に廃止しない
- ただし uncertainty が高い案件、探索案件、体験案件では runtime default を強める
- `Value Alignment Loop` を強く回す案件では、recent confirmation window と mismatch state を runtime で持つことを標準寄りとする

## Out Of Scope For v1.6

- full UX testing platform
- automated visual quality scoring
- computer-vision-based expectation matcher
- hosted review gallery
- rich interactive prototype generator
- domain-specific style guides for every product class

## Non-Goals

1. AOF 自体が taste judgment を自動化すると主張しない
2. screenshot proof だけで experience truth が十分だと主張しない
3. visual / spatial product だけを特別扱いする release にしない
4. reviewer UI や gallery tooling をこの版で完成させない
5. experience-validation を heavyweight enterprise gate にしない
6. interview を長い面談中心にしない

## Release Gates

### Gate 1: v1.5 Integrity Is Preserved

- trust assumptions remain coherent
- capability / independence / review cadence docs are not weakened
- expectation validation is added without contradicting prior governance rules

### Gate 2: Goal Layers And Value Alignment Loop Are Explicit

- docs define `North Star Goal / Current Operating Goal / Next Value Slice`
- docs define repeated confirmation, recent confirmation window, and alignment pulse
- docs connect the loop to mismatch handling and scale-up checks

### Gate 3: Experience Validation Is Explicit

- docs define `experience-validation` as a real phase
- docs define minimum outputs and exit rule
- docs state that downstream confidence should not grow without live experience evidence

### Gate 4: Experience Steward Is Explicit

- docs define the role and its responsibilities
- docs show how it complements, rather than replaces, existing governance actors
- docs connect the role to mismatch detection and proof review

### Gate 5: Fidelity Split And Expectation-Mismatch Are Operational

- docs distinguish viewer fidelity from editor / authoring fidelity
- docs explain why they must not be collapsed into one backend judgment
- docs explain when mismatch should be raised
- docs connect it to reframe / reprioritize / re-architecture loops

### Gate 6: Proof Categories And Runtime Default Guidance Are Explicit

- docs define proof categories clearly
- docs distinguish specification proof from product-feel proof
- docs show how screenshot / interaction proof participate in review
- docs explain when uncertainty should push usage toward runtime default

## Deferred To v1.7+

- runtime-enforced experience-validation checkpoints
- richer visual review dashboards
- automated screenshot diff workflows
- domain-specific experience-evaluation packs
- quantitative experience confidence models
- deeper user-interview ontology beyond repeated confirmation

## Current Intended Shape

`v1.6` は、AOF を “more documentation” の方向に進める版ではない。  
重点は、

- 仕様上は正しく見えるのに、体験としてはズレている
- architecture reasoning は妥当だが、authoring feel が違う
- product boundary は守れているが、product reality がずれている
- next slice は出ているが、North Star との接続が弱い

という failure mode を早く止めることにある。

したがって `v1.6` は、AOF の governance strengths を維持したまま、  
**live experience truth と value alignment loop を release gate に組み込む版** とする。
