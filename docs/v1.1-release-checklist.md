# AOF v1.1 Release Checklist

`v1.1` を release-ready と判定する前に確認する checklist。  
この文書は [docs/v1.1-release-definition.md](./v1.1-release-definition.md) の gate を、repo 内の evidence と実行コマンドに落としたものとする。

## How To Use This Checklist

1. gate ごとに必要 evidence を確認する
2. local evidence は repo 内の fixture、docs、test、smoke を正本とする
3. missing evidence がある場合は `v1.1 ready` と言わない

`v1.1` の役割は analytics 完成ではなく、measurement foundation と operational policy を成立させることである。

## Evidence Classes

### Local Evidence

- committed example sessions
- schema and runtime sources
- `node ./scripts/test-runner.js`
- `node ./scripts/smoke.js`
- operator-facing docs

### Readiness Evidence

`v1.1` では次を揃えて readiness を主張する。

- current commit SHA
- green local test result
- green smoke result
- example session showing `stage_transitions[]`
- example session showing `outcome-report` writeback
- docs for runtime usage policy and concurrency policy

Tag 前の release hygiene として、local evidence に加えて current candidate commit の GitHub Actions `CI` run が green であることも確認する。

## Blocking Conditions

次のどれかが残る場合、`v1.1 ready` とは言わない。

1. local test runner が赤い
2. smoke が赤い
3. committed example session が schema-valid でない
4. `outcome-report` writeback evidence が repo 内に存在しない
5. framing-only / runtime-on / serial-only policy が docs で読めない
6. tag 前の current candidate commit で `CI` run の green 確認が取れていない
