# AOF v1.10 Release Definition

AI Organization Framework を `v1.10` として更新・リリースするための基準を定義する。

`v1.10` の主題は、`v1.9` で成立した self-hosting cadence runtime を  
**人間から見える operating surface と topology-aware GitHub operations policy** に押し上げることにある。

## Position

`v1.9` が cadence runtime を self-hosting loop の中で動かせるようにした版であるなら、  
`v1.10` は **その cadence state を visibility contract に載せ、self-hosting 例外を general default にしない GitHub operations rule を固定する版** である。

重点は次である。

- cadence timing / dispatch / scheduler profile を human visibility contract に載せる
- `visibility-serve` が cadence summary を読めるようにする
- self-hosting topology と managed-project topology を明示的に分ける
- GitHub Actions cadence binding が self-hosting repo の topology-specific choice であることを固定する
- `.aof/**` cadence writes と product CI の分離を明示する

## v1.10 Goal

`v1.10` の主目標は次である。

1. cadence runtime state が raw `.aof` artifact だけでなく visibility contract からも読めること
2. operator が cadence timing / dispatch / scheduler profile を viewer で把握できること
3. self-hosting repo の GitHub operations rule を managed project の default と切り分けること
4. cadence automation write target が topology-dependent であることを AOF docs 全体で一貫させること

## In Scope For v1.10

### Cadence Visibility Contract

`status_card` は cadence summary fields を optional に持てる。

最低限:

- `cadence_timing_state`
- `cadence_scheduler_state`
- `cadence_dispatch_state`
- `cadence_scheduler_profile`
- `cadence_next_check_at`
- `cadence_reason`

これらは raw cadence artifact の全文ではなく、human-facing summary として扱う。

### Viewer-Level Cadence Readability

`visibility-serve` は cadence summary fields が status payload にある場合、

- project state
- overview cards
- hero summary

のどこかで人間が読める形に出す。

ここで重要なのは advanced dashboard を作ることではなく、  
**existing visibility contract の中に cadence state が入ること** である。

### Topology-Aware GitHub Operations

GitHub / branch strategy は topology-dependent であることを固定する。

必要な 2 topology:

1. `self-hosting topology`
2. `managed-project topology`

minimum rule:

- self-hosting:
  - cadence may write `.aof/**` to repository `main`
- managed-project:
  - cadence must not push `.aof/**` directly to product `main`
  - default write target is `aof/state` branch or equivalent separated channel

### Self-Hosting Binding Hardening

current self-hosting GitHub Actions cadence binding について、少なくとも次を明示する。

- `.aof/**` cadence state commit は full product CI と分離する
- workflow comment / docs で self-hosting topology 前提を明示する
- commit message は dispatch/tick summary を追跡できる
- stage 対象は cadence-updated surfaces に絞る

## Out Of Scope For v1.10

- full visibility producer automation from live `.aof` state
- advanced cadence dashboard pane
- cross-repo scheduler federation
- full managed-project bootstrap templates
- multi-session cadence board

## Non-Goals

1. `v1.9` cadence runtime claimを壊さない
2. self-hosting workflow を managed-project default に昇格させない
3. cadence artifact を viewer のみの専用 state に分岐させない
4. `.aof/**` cadence writes によって product CI を毎回起動させない

## Release Gates

### Gate 1: v1.9 Cadence Runtime Integrity Is Preserved

- cadence timing / schedule / dispatch / profile artifacts remain intact
- `v1.10` additions do not weaken `v1.9` runtime-backed cadence loop

### Gate 2: Visibility Contract Can Carry Cadence Summary

- cadence summary fields are defined in `human-visibility-output-formats.md`
- those fields are clearly optional and backward-compatible

### Gate 3: visibility-serve Reads Cadence Summary

- viewer can render cadence summary from status payload
- behavior is covered by tests or equivalent operator verification

### Gate 4: GitHub Operations Model Is Topology-Aware

- self-hosting vs managed-project distinction is explicit
- managed-project default forbids direct cadence pushes to product `main`
- recommended write targets are documented

### Gate 5: Self-Hosting GitHub Binding Is Hardened

- `.aof/**` cadence commits do not trigger full CI
- cadence workflow is scoped and concurrency-safe enough for current use
- workflow and docs clearly state that the binding is topology-specific

### Gate 6: Self-Audit And Task Evidence Reflect The New Boundary

- at least one self-audit artifact records the cadence-visibility closure
- the next task after closure is explicit
- current AOF task/goal state reflects the new gap

## Deferred To v1.11+

- live `.aof` -> visibility output producer automation
- managed-project starter examples with `aof/state`
- cadence pane specialized UI
- external scheduler profile templates beyond current binding guidance

## Current Intended Shape

`v1.10` は、AOF を

**“cadence runtime exists, but operators must read raw artifacts”**

から

**“cadence runtime is visible through the existing human visibility contract, and GitHub operations policy is topology-safe”**

へ進める版とする。
