# AOF v3.8.0 Release Notes

- target version: `v3.8.0`
- current package version: `3.8.0`
- status: signed off
- release note revision commit SHA: pending

## Summary

`v3.8.0` is the release that turns runtime situation judgment into a compact operator-facing answer surface.

`v3.7.0` made the runtime truthful about the current frontier.  
Running that runtime exposed the next problem:

- the current answer was still spread across multiple surfaces
- Mission Control was useful, but not the best primary operator path
- the operator still had to reconstruct `what / why / blocked / next`

`v3.8.0` addresses that by adding a first-class operator briefing layer above situation assessment.

## Highlights

### 1. Operator Brief Becomes First-Class

`operator-brief` now emits one packet that answers:

- what is happening now
- why that is the current state
- what is blocked
- what should happen next

### 2. The Brief Stays Derived From Runtime Truth

The brief is not a second truth model.  
It is derived from canonical runtime artifacts including:

- active release state
- current goals
- current frontier task
- runtime situation assessment

### 3. Visibility Export Now Includes The Operator Brief

`visibility-export` now writes:

- `status-card.json`
- `timeline-feed.json`
- `flow-snapshot.json`
- `mission-control.json`
- `operator-brief.json`

This keeps Mission Control available while making the operator brief the faster explanation path.

### 4. Mission Control Normalization Bug Is Fixed

`mission_control` no longer treats the active release `3.7.0` as if it had no track and therefore no longer keeps obsolete chain-gap blockers on the current operator path.

## Boundary

`v3.8.0` does not claim:

- richer observability authority
- broader analytics authority
- new execution authority
- viewer-first operator flow

It claims a narrower thing:

**AOF can compress truthful runtime judgment into one compact operator-facing brief without breaking artifact alignment.**
