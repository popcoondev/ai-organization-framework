# AOF v3.6 Release Definition

`v3.6.0` の主題は、
**runtime truth から導出される bounded Mission Control visibility layer**
を release-grade にすることである。

## Position

`v3.5.0` は CLI surface を routing しやすくした。  
しかし runtime を実際に回すと、次の operator friction が残った。

- status / timeline / flow が分かれていて mission 状態を一画面で読みにくい
- artifact lineage は runtime にあるが main operator path で辿りにくい
- blockers と recommended next action が unified されていない
- 今なにが起きているかは見えても、なぜそうなったか / 次に何をすべきかは再構成が必要

したがって `v3.6.0` でやるべきことは decorative dashboard 追加ではない。  
やるべきことは **Mission Control を runtime-derived truth surface として bounded に成立させること** である。

## v3.6 Goals

1. mission overview を canonical runtime state から導出する
2. artifact lineage を one-surface で読める causal chain として出す
3. blockers を runtime-derived summary として表示する
4. recommended next action を canonical artifacts から導出する
5. Mission Control の stage transition が benchmark で再現・検証できるようにする

## Out Of Scope

- decorative pixel-office layer
- live role-state or council-state claims that current runtime artifacts cannot derive truthfully
- Discovery authority expansion
- Need Validation boundary changes
- backend-specific autonomy claims
- any second source of truth outside canonical artifacts

## Release Gates

### Gate 1: Mission Overview Is Canonical

- one Mission Control overview exists on the operator path
- current mission, release, operating goal, next value slice, and current runtime stage are derived from runtime state

### Gate 2: Artifact Lineage Is Legible

- operators can see what artifact chain produced the current mission state
- lineage is derived from governed artifacts or narrow summaries of them

### Gate 3: Blockers And Next Action Are Unified

- the operator can identify blockers from Mission Control alone
- the operator can identify the recommended next action from the same surface

### Gate 4: Stage Transition Is Truthful

- Mission Control promotes from baseline to discovery handoff
- Mission Control promotes from discovery handoff to planning-ready after Need Validation
- Mission Control promotes to implementation-ready when implementation work is opened
- these transitions are benchmarked from a real runtime artifact chain

### Gate 5: Boundary Remains Honest

- Mission Control does not invent new authority
- Discovery remains evidence
- Need Validation remains approval authority
- visibility remains additive and artifact-derived

## Intended Outcome

After `v3.6.0`, the runtime should be able to say, and verify, one coherent thing:

```text
this is the current mission
this is the artifact chain that produced it
this is what is blocked
this is what should happen next
and those statements are derived from runtime truth rather than a separate dashboard story
```
