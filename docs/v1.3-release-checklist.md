# AOF v1.3 Release Checklist

`v1.3` を release-ready と判定する前に確認する checklist。  
この文書は [docs/v1.3-release-definition.md](./v1.3-release-definition.md) の gate を、repo 内の evidence と concept-validation artifact に落としたものとする。

## How To Use This Checklist

1. gate ごとに必要 evidence を確認する
2. `v1.3` では operating-model docs と concise validation artifact を正本とする
3. missing evidence がある場合は `v1.3 ready` と言わない

`v1.3` の役割は runtime automation を増やすことではなく、Discovery-driven team assembly と mission control を固定することである。

## Blocking Conditions

次のどれかが残る場合、`v1.3 ready` とは言わない。

1. Discovery が acceleration model として docs で読めない
2. team assembly が sizing からどう出るか読めない
3. operating goal / autonomy budget / escalation boundary が読めない
4. mission control loop と drift response が読めない
5. evolving-task を対象にした concise validation artifact が存在しない
