# AOF v3.8 Release Definition

`v3.8.0` の主題は、
**runtime situation assessment を compact operator briefing layer にすること**
である。

## Position

`v3.7.0` で runtime は current frontier と truth conflict を診断できるようになった。  
しかし operator が状況を理解するには、まだ複数の surface を跨いで読む必要があった。

- `situation-assess` は truthful だが compact answer ではない
- `roadmap-status` は current frontier を示せるが explanation surface ではない
- Mission Control は additive だが primary path にすると viewer-first へ戻る
- operator が `why / blocked / next` を 1 つの答えとして受け取る surface がまだない

したがって `v3.8.0` でやるべきことは richer viewer ではない。  
やるべきことは **runtime judgment を one-packet operator brief に圧縮すること** である。

## v3.8 Goals

1. a first-class `operator-brief` command exists
2. the briefing packet answers what is happening, why, what is blocked, and what should happen next
3. the briefing packet stays derived from situation assessment and other canonical artifacts
4. `visibility-export` emits the same operator brief alongside Mission Control
5. Mission Control remains additive rather than mandatory for basic runtime comprehension

## Out Of Scope

- decorative viewer expansion
- richer role-state or council-state live views
- broader organization analytics claims
- new execution authority
- a second operator truth model maintained outside canonical artifacts

## Release Gates

### Gate 1: Operator Brief Exists

- a dedicated command emits one compact operator briefing packet
- the packet answers current state, causal basis, blockers, and next action

### Gate 2: Briefing Stays Truthful

- the operator brief follows the same frontier as situation assessment
- the recommended action in the brief matches the live frontier task
- the brief does not reintroduce stale shipped work

### Gate 3: Viewer Is No Longer The Main Operator Path

- the operator can answer basic runtime questions from the briefing packet alone
- Mission Control remains additive instead of becoming the required explanation path

### Gate 4: Mission Control And Briefing Stay Aligned

- `visibility-export` emits `mission_control` and `operator_brief` from the same truth source
- Mission Control does not keep obsolete blockers caused by release-track normalization bugs

### Gate 5: Boundary Remains Honest

- briefing remains artifact-derived and backend-neutral
- Discovery remains evidence
- Need Validation remains approval authority
- no new execution authority is claimed

## Intended Outcome

After `v3.8.0`, the runtime should be able to tell the operator one coherent thing:

```text
this is what is happening now
this is why that is the current state
this is what is blocked
this is what should happen next
and that answer is derived from canonical runtime artifacts
```
