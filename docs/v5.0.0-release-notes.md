# AOF v5.0.0 Release Notes

- target version: `v5.0.0`
- current package version: `5.0.0`
- status: signed off

## Summary

`v5.0.0` introduces **Skillful Actor Runtime**.

This release moves AOF from:

```text
roles can be shown in the organization
```

to:

```text
actors can be assigned through skill, capability, resource, policy, review, and HRI evidence
```

The release does not claim broad autonomous execution.  
It claims a narrower and more important capability: AOF can make actor assignment judgeable before execution and falsifiable after review.

## Highlights

### 1. Actor Skill Packet Contract

The new actor skill packet contract defines why an actor or role should do a piece of work.

It includes:

- objective
- actor / role / team assignment
- required skill refs
- capability-fit evidence
- resource refs
- policy refs
- output contract
- review criteria
- blocker semantics
- Human Recognition Interface projection

Canonical docs and schema:

- `docs/v5-actor-skill-packet-contract.md`
- `schemas/aof-actor-skill-packet.schema.json`

### 2. Governed Assignment Evaluation

AOF can now evaluate whether an actor assignment is selected, degraded, blocked, or requires evidence.

This prevents weak actor selection from becoming invisible execution momentum.

Canonical runtime:

- `src/commands/actor-assignment-evaluation-record.js`
- `schemas/aof-actor-assignment-evaluation.schema.json`

### 3. Resource And Policy Execution Gate

Selected assignments now pass through an execution gate that reads resource claim and policy evaluation evidence.

The gate can return:

- `allowed`
- `degraded`
- `blocked`
- `requires-council-review`

Canonical runtime:

- `src/commands/actor-execution-gate-record.js`
- `schemas/aof-actor-execution-gate.schema.json`

### 4. Negative Skillful Actor Benchmarks

`v5.0.0` adds benchmark coverage that rejects fake skillful success.

The benchmark checks:

- missing skill evidence
- weak actor assignment
- stale release state
- policy-bypassed allocation

Canonical runtime:

- `src/commands/skillful-actor-benchmark.js`
- `schemas/aof-skillful-actor-benchmark.schema.json`

### 5. Human Recognition Interface Projection

Skillful Actor state can now be projected into the Human Recognition Interface from canonical artifacts.

The projection carries:

- actor label
- current assignment
- skill summary
- confidence
- blocker state
- council review need
- speech bubble
- next action

Canonical runtime:

- `src/commands/skillful-actor-hri-projection.js`
- `schemas/aof-skillful-actor-hri-projection.schema.json`
- `.aof/artifacts/skillful-actor/hri-projections/SAHRI-TASK-054-PROOF.json`

## Boundary

`v5.0.0` does not change:

- Discovery authority
- Need Validation authority
- council approval semantics
- backend-neutral runtime position
- Human Recognition Interface authority boundary

The release adds governed actor skillfulness. It does not claim that AOF can autonomously run arbitrary organizations without human and artifact governance.
