## Summary

`v1.5.0` is planned as the release that makes AOF's **trust assumptions and operating assumptions** explicit.

Where `v1.4.0` made runtime evidence visible to humans, `v1.5.0` is intended to explain:

- when routing can be trusted
- when Council of Three disagreement is only nominal
- when Discovery / Clarification quality is too weak to proceed
- when human review must happen, not just may happen
- when structured output or context reading should escalate instead of silently continuing

## Planned Scope

- routing capability assumptions
- council independence and execution-pattern limits
- clarification / discovery quality floor
- human review cadence by usage level
- structured-output fallback limits
- context completeness guidance

## Planned Non-Goals

- stronger automation by itself
- richer orchestration backend
- probabilistic confidence calibration
- full reviewer workforce modeling

## Non-Core But Shipped

`v1.5` の主題は trust assumptions であり、viewer 拡張ではない。  
ただし `v1.4` までで追加された human visibility surface は preserved foundation として維持する。

- `status_card`
- `timeline_feed`
- `flow_snapshot`
- `visibility-serve`
