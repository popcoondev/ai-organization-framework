# Next Major Version Roadmap

## North Star

AOF should become an AI Organization Operating System that can establish, operate, govern, measure, improve, split, merge, and archive project organizations.

## Phase 1: Organization Planning Proof

Goal:

- Demonstrate that AOF 2.1 can plan a next major version using its own organization model.

Deliverables:

- requirements analysis
- organization manifest
- team charters
- contracts
- ADR
- roadmap
- release plan

Exit criteria:

- Council approval is recorded.
- All success criteria in the mission are represented by artifacts.

## Phase 2: Organization Runtime Backbone

Goal:

- Make organization artifacts actionable by runtime commands.

Candidate deliverables:

- `aof organization-status`
- `aof team-charter`
- `aof contract-register`
- `aof dependency-graph`
- `aof adr-record`

Exit criteria:

- Organization state can be inspected without reading raw JSON.
- Contracts and dependencies can be listed and validated.

## Phase 3: Council And Team Execution Loop

Goal:

- Move from static organization manifest to repeatable council/team operating loop.

Candidate deliverables:

- council review packet
- role result contract
- team output contract
- judge loop artifact

Exit criteria:

- Work can flow from team output to council judgment to next task without relying on informal notes.

## Phase 4: Measurement And Lifecycle

Goal:

- Make metrics and lifecycle more than placeholders.

Candidate deliverables:

- project health snapshot
- risk / quality / cost metric register
- lifecycle transition record
- archive / split / merge decision template

Exit criteria:

- Organization lifecycle transitions are auditable.
- Metrics have owners and update cadence.

## Phase 5: Execution Backend Integration

Goal:

- Allow AOF organization plans to run on concrete AI execution backends without becoming backend-specific.

Candidate deliverables:

- Codex parent/child orchestration guide
- Claude workflows backend guide
- backend-neutral role assignment contract

Exit criteria:

- AOF can describe organization intent independently of whether Codex, Claude, GPT, human, or tool resources execute roles.
