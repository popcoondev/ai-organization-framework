# AOF v1.4 Release Definition

AI Organization Framework を `v1.4` として更新・リリースするための基準を定義する。

`v1.4` の主題は、runtime の記録を **人間が状況把握できる可視化層** に変えることにある。

## Position

`v1.4` は dashboard 拡張版ではなく、**Human Visibility Layer** の版である。

`v1.3` で discovery-driven team assembly と mission control を operating model として固定したうえで、

- 人間が今どこにいるか
- 誰が何を決めたか
- なぜ次にそれをやるのか

を runtime evidence から読めるようにする。

## v1.4 Goal

`v1.4` の主目標は次である。

- runtime event / log の minimum contract を定義する
- human-facing `status / timeline / flow` view を定義する
- `static AOF` と `runtime-used AOF` の見分けを人間が即座にできるようにする
- 人間が逐一質問しなくても current state を把握できる control surface を持つ

## In Scope For v1.4

### Runtime Event / Log Contract

- session, signal, checkpoint, decision の minimum event shape
- human-facing projection に必要な fields を定義する
- log source は text でも DB でもよいが、同じ projection が可能であること
- first concrete contract can be [docs/candidate-lifecycle-event-contract.md](./candidate-lifecycle-event-contract.md)

### Status View

minimum fields:

- current phase
- current operating goal
- owner
- open signals
- next checkpoint
- current usage level

### Timeline View

minimum fields:

- when
- who
- what changed
- why it changed
- what became next

### Flow View

minimum elements:

- phase transitions
- signal emission
- reframe / rescope / reprioritize events
- current position marker

### Static vs Runtime-Used Distinction

- 人間が「これは static AOF か runtime-used AOF か」を見分けられる
- runtime evidence が無ければ、それを明示する

## Out Of Scope For v1.4

- full hosted observability service
- rich GUI productization
- complex analytics dashboards
- autonomous remediation
- multi-tenant operations backend

## Non-Goals

1. 可視化の見た目を先に豪華にしない
2. event log の量を増やすこと自体を目的にしない
3. 人間向け visibility を front-AI 専用機能にしない
4. analytics と control surface を混同しない

## Release Gates

### Gate 1: v1.3 Integrity Is Preserved

- mission control vocabulary remains coherent
- short kickoff と longer-running operation の両方で visibility が説明できる

### Gate 2: Runtime Event Contract Is Explicit

- minimum event/log fields が定義されている
- text / DB の違いに依らず human-facing projection が可能

### Gate 3: Human Status View Is Explicit

- current phase / goal / owner / open signals / next checkpoint が読める

### Gate 4: Timeline And Flow Views Are Explicit

- judgment history と flow transitions の両方が定義されている

### Gate 5: Static vs Runtime-Used Distinction Is Visible

- runtime evidence の有無が人間から見て明確

## Deferred To v1.5+

- richer visual UI kits
- stronger analytics overlays
- automated explanation generation
- deeper runtime orchestration

## Current Intended Shape

`v1.4` は、AOF runtime を人間に見える operating surface に変える版である。  
重点は「大量ログ」ではなく、

- 今どこにいるか
- 誰が何を決めたか
- 次に何が起きるか

をグラフィカルに把握できることに置く。
