# AOF v1.8 Release Definition

AI Organization Framework を `v1.8` として更新・リリースするための基準を定義する。

`v1.8` の主題は、AOF が project memory, task ledger, and self-audit loop を  
`.aof/` 内に自己保有することにある。

## Position

`v1.7` が `AI Orchestrator` と orchestrated multi-session governance model を formalize した版であるなら、  
`v1.8` は **AOF-native project memory** を formalize する版である。

重点は次である。

- `.aof/tasks/` を source of truth とする
- `.aof/goals/` を goal state projection として定義する
- `.aof/context/threads/` と `.aof/prompts/` を追加して thread work と template を収める
- `Alignment Pulse` に `Task Triage` を組み込む
- `Archivist` に task / thread archive の責務を追加する
- AOF 自体に `Framework Self-Audit` を適用する

## v1.8 Goal

`v1.8` の主目標は次である。

1. AOF-native task management を定義する
2. `.aof/` directory structure を canonical にする
3. `Task Triage` を `Alignment Pulse` に組み込む
4. `Archivist` の scope を task / thread archive まで拡張する
5. `Framework Self-Audit` を定義する

## In Scope For v1.8

### AOF-Native Task Management

- `.aof/tasks/` を唯一の task ledger として定義する
- minimum lifecycle:
  - `open/`
  - `assigned/`
  - `done/`
  - `archived/`
  - `retired/`
- task の source of truth は AOF 内とし、GitHub Issues は mirror でしかない

### Task Ownership Model

`v1.8` では task assignment を `single session owner` ではなく、  
**orchestrator-centered ownership** として扱う。

最低限、task は次を読める必要がある。

- `orchestrator_session_id`
- `assigned_session_ids[]` or equivalent child-session references
- `related_decision_record_id`
- `operating_goal_ref`

つまり、task は 1 child thread に閉じず、Orchestrator が canonical owner となる。

### Goals Projection

`.aof/goals/` は当面、new canonical source ではなく **materialized projection** として扱う。

意味:

- canonical ownership は `State Ownership Model` のまま
- `North Star Goal`
- `Current Operating Goal`
- `Next Value Slice`

を human-readable / machine-readable に投影する

この版では、goal projection の配置と同期ルールを定義するが、  
session canonical state を捨てて `goals/` 側へ完全移行することは主張しない。

### Task Triage In Alignment Pulse

`Alignment Pulse` の確認項目に `Task Triage` を追加する。

最低限確認する問い:

1. `.aof/tasks/open/` の中で Current Operating Goal に関係するものは何か
2. stale task を retire 候補として扱うか
3. 次の value slice 前に確認すべき open task は何か

### Archivist Expansion

`Archivist` は context hygiene だけでなく、次を担う。

1. `.aof/tasks/done/` から `.aof/tasks/archived/` への移送
2. stale task の triage 準備
3. `.aof/context/threads/` の完了出力整理
4. session completion 後の thread-local output archive

### Framework Self-Audit

AOF 自体に Discovery を適用する。

最低限:

- periodic or release-based self-audit
- scope:
  - lifecycle coverage
  - external dependency drift
  - role completeness
  - schema reflection
  - document freshness
- output:
  - `.aof/tasks/open/TASK-*.json`
  - self-audit summary artifact

## Data Model Notes

### Task Schema

task schema には少なくとも次の metadata を持たせる。

- `task_id`
- `title`
- `status`
- `origin`
- `orchestrator_session_id`
- `assigned_session_ids`
- `related_decision_record_id`
- `operating_goal_ref`
- `created_at`
- `updated_at`
- `assigned_at optional`
- `done_at optional`
- `retired_at optional`
- `last_triaged_at optional`
- `triage_notes optional`

理由:

- stale 判定
- archive timing
- retire reasoning
- task triage cadence

を later inference ではなく explicit metadata で扱いたいからである。

### Goal Projection Structure

goal projection は次のような分割を許容する。

- `.aof/goals/north-star.json`
- `.aof/goals/operating-goal.json`
- `.aof/goals/next-value-slice.json`

ただし、projection であることを忘れない。  
canonical authority は引き続き Orchestrator 側 state ownership にある。

### Prompt Storage Rule

`.aof/prompts/` は **template storage** を主目的とする。

区別:

- `prompts/`: versioned templates
- `context/threads/` or equivalent runtime area: realized thread-specific prompt/output artifacts

この版では template と runtime expansion を混同しない。

## Runtime And Viewer Impact

### Runtime

`v1.8` は runtime に近づくが、まだ full orchestration daemon は要求しない。

minimum required scope:

- task schema
- goals projection structure
- archive / triage / self-audit model

つまり、runtime command の全面更新はこの版の必須ではない。

### Viewer

task ledger や multi-session memory の viewer 拡張は有用だが、`v1.8` の required scope ではない。  
viewer integration は `v1.9+` に送ってよい。

## Out Of Scope For v1.8

- GitHub Issues 双方向同期
- task auto-retire automation
- task UI / dashboard expansion
- full orchestration daemon
- multi-session visibility dashboard
- cross-orchestrator coordination

## Non-Goals

1. external issue tracker を source of truth に戻さない
2. `.aof/goals/` を急いで唯一の canonical source にしない
3. task archive に毎回 Human 承認を必須化しない
4. prompt template と runtime thread artifacts を混同しない
5. viewer expansion を core gate に入れない

## Release Gates

### Gate 1: v1.7 Integrity Is Preserved

- orchestrator governance model is not weakened
- topology / independence / state ownership semantics are preserved

### Gate 2: AOF-Native Task Management Is Defined

- `.aof/tasks/` structure and lifecycle are defined
- task origin and transition authority are explicit
- task ownership is orchestrator-centered
- `schemas/aof-task.schema.json` exists

### Gate 3: Alignment Pulse Task Triage Is Defined

- `Task Triage` is part of alignment pulse
- human-facing triage prompts are defined
- stale task reading is defined

### Gate 4: Archivist Scope Is Extended

- Archivist includes task and thread archive responsibilities
- Archivist trigger conditions are explicit

### Gate 5: .aof/ Directory Structure Is Canonical

- `docs/aof-directory-structure.md` exists
- directories have purpose and retention rules
- `schemas/aof-goals.schema.json` or equivalent projection schema exists

### Gate 6: Framework Self-Audit Is Defined

- `docs/framework-self-audit.md` exists
- self-audit scope, trigger, and output are explicit
- at least one self-audit evidence artifact exists

## Deferred To v1.9+

- GitHub Issues export/import
- task visualization UI
- automatic task retirement heuristics
- multi-session viewer surfaces
- prompt update governance automation

## Current Intended Shape

`v1.8` は、AOF を external tool-assisted framework から、  
**AOF-native task memory, goal projection, and self-audit loop を持つ operating system** に近づける版とする。
