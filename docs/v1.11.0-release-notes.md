## Summary

`v1.11.0` is planned as the release that formalizes AOF’s Codex parent/child orchestration model under current, real Codex capabilities.

Where `v1.10.0` made cadence runtime state human-visible and GitHub operations topology-aware, `v1.11.0` is intended to explain and demonstrate:

- how Human-started parent Codex orchestration should work
- how child role execution should be modeled
- what result contract children should return
- how parent join / aggregate should be represented
- where the boundary still lies between cadence runtime and future autonomous Codex startup

## Planned Scope

- Codex parent/child orchestration model
- role result artifact contract
- parent join artifact contract
- explicit capability boundary for current Codex-based orchestration

## Planned Non-Goals

- autonomous Codex daemon
- scheduler-triggered Codex startup bridge
- runtime implementation of all parent/child orchestration artifacts
- managed-project bootstrap examples

## Current Reading

- cadence runtime and cadence visibility remain intact from `v1.10.0`
- current repo now has a dedicated Codex parent/child orchestration model document
- child role result and parent join are both formalized as docs + JSON schemas
- CI is maintainer-confirmed green and the release candidate is sign-off ready
