# AOF v3.3.0 Release Notes

- target version: `v3.3.0`
- current package version: `3.3.0`
- status: signed off
- release note revision commit SHA: `to be fixed by the release tag`

## Summary

`v3.3.0` is the release that makes Discovery evidence a governed upstream contract rather than a research-only side layer.

`v3.2.0` already required Need Validation before project creation.  
`v3.3.0` extends that contract so the runtime can now distinguish between:

- exploration that is still too weak to hand off
- discovery evidence that is strong enough to narrow a need
- a handoff that is explicit enough to enter Need Validation

The claim is not "Discovery is autonomous."  
The claim is "upstream evidence quality is now part of pre-project governance."

## Highlights

### 1. Discovery Contract Is Now Release-Grade

`v3.3.0` fixes the minimal governed artifact set before Need Validation:

- discovery question set
- assumption map
- anomaly log
- discovery judgment packet
- discovery handoff record

`breakthrough-pattern-record` remains optional as a strengthening input.

### 2. Discovery Judgment Is First-Class

Discovery no longer ends in prose-only recommendations.

The governed statuses remain:

- `continue-exploration`
- `pivot`
- `synthesize-handoff`
- `stop`

This makes blocked exploration, not only successful handoff, part of the contract.

### 3. Discovery Handoff Is Benchmarked

`discovery-handoff-benchmark` adds a narrow benchmark family:

- `DH-001` mandatory discovery artifacts presence
- `DH-002` discovery judgment validity
- `DH-003` handoff packet completeness
- `DH-004` handoff-to-need-validation linkage
- `DH-005` blocked upstream evidence representation

The benchmark is intentionally narrow.  
It checks contract completeness, not broad discovery quality scoring.

### 4. The Self-Hosting Repo Now Contains A Canonical Planning-Grade Chain

`TASK-037` / `SESS-MQGOLKU3-IK9XEE` is the committed example showing:

```text
Discovery
  -> discovery judgment
  -> discovery handoff
  -> Need Validation
  -> project charter
  -> planning
```

Direction setting itself was run through the new contract rather than being left as prose-only planning.

## Boundary

`v3.3.0` does not claim:

- direct project creation from Discovery alone
- full Discovery Council productization
- autonomous market research execution
- backend-specific multi-agent expansion
- human-review removal

It claims a stronger upstream evidence governance contract.

## Verification

Release verification should include:

- `node --test test/runtime.test.js`
- `node ./src/cli.js organization-verify --project .`
- `node ./src/cli.js discovery-handoff-benchmark --project .`
- `npm test -- --runInBand`
- `npm run smoke`

Sign-off was recorded on `2026-06-17` with one accepted limitation:

- the local Node test harness still shows intermittent `node --test` / `npm test` import instability even when direct suite import, `organization-verify`, `discovery-handoff-benchmark`, and `npm run smoke` are green

## Honest Weaknesses

The release still has real follow-up work:

- discovery evidence is governed more clearly than before, but not fully productized as a complete Discovery Council runtime
- benchmark coverage is intentionally narrow and should stay contract-focused
- terminology still needs careful operator wording so discovery evidence is not confused with approval authority
