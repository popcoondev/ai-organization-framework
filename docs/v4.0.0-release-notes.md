# AOF v4.0.0 Release Notes

- target version: `v4.0.0`
- current package version: `4.0.0`
- status: signed off
- release note revision commit SHA: pending

## Summary

`v4.0.0` is the release that turns the truthful `v3.9.0` viewer path into a one-screen **Human Recognition Interface**.

`v3.9.0` made the operator path truthful about:

- what is happening now
- what changed
- where the work sits in the release tree
- why the current answer is true
- how to open the viewer from one runtime path

Running that truth in a real viewer exposed the next bottleneck:

- the operator could still reconstruct the story only by reading multiple panels
- live organization motion was not immediately legible
- packet truth and product surface were still too close together
- runtime-backed status and live-loop status were not yet the first thing a human recognized

`v4.0.0` addresses that by promoting the product target from a recognition packet artifact to a **Human Recognition Interface powered by the Human Recognition Packet**.

## Highlights

### 1. Human Recognition Interface Becomes First-Class

The viewer is now framed as a Human Recognition Interface rather than a generic visibility viewer.

The intended operator question becomes:

- who is in the organization now
- what are they doing
- who is judging
- what changed
- what is blocked
- what should happen next
- is this answer really runtime-backed

### 2. Artifact -> Packet -> Interface Mapping Becomes Canonical

`v4.0.0` defines the canonical mapping from:

- runtime artifacts
- to Human Recognition Packet
- to character / roadmap / timeline / blocker / next-action surfaces

This mapping is documented explicitly rather than being left implicit in UI code alone.

### 3. Runtime-Backed Fact Becomes Visible

The interface now surfaces:

- runtime-backed pass / incomplete state
- last runtime execution time
- command chain
- refreshed artifact refs

This makes direction / review / self-review claims visibly weaker when no runtime proof exists.

### 4. Live Organization Loop Becomes Readable

The `TASK-046` frontier is backed by a live runtime loop that can be shown as:

- council invocation
- council seat judgments
- workforce assembly
- council review
- orchestrator return

This is the core product move in `v4.0.0`: not just more state, but more recognizable organizational motion.

After the release cut, the runtime truth now advances cleanly into:

- shipped trunk: `v4.0`
- active branch: `v5.0`
- live frontier: `TASK-047`

This matters because the Human Recognition Interface is not only a release artifact. It remains truthful after the release transition.

### 5. Human Recognition Replaces Static Visibility As The Product Test

The release does not claim decorative game UI.

It claims a narrower and stronger thing:

**AOF can make live organization truth instantly recognizable to a human on one screen without inventing a second truth model.**

## Boundary

`v4.0.0` does not claim:

- decorative character theater without artifact backing
- invented speech bubbles
- new approval authority
- Discovery or Need Validation boundary changes
- autonomy expansion justified only by visualization

It claims:

**the runtime can turn canonical artifacts into a one-screen Human Recognition Interface that keeps truth, loop state, and runtime-backing legible.**
