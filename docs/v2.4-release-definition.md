# AOF v2.4 Release Definition

`v2.4.0` の主題は、`v2.3.0` までに operator-facing に見えるようになった organization runtime を、execution packet と judgment artifact の連鎖まで first-class に進めることである。

## Position

`v2.3.0` は organization / contract / dependency / decision / roadmap / audit を operator surface として固定した。  
`v2.4.0` は、その surface の上で実際の work product がどう流れるかを、backend-neutral な packet contract として固定する版である。

ここでやるのは autonomous runtime claim ではない。  
role output、team aggregation、council judgment、execution lineage を ad hoc note ではなく schema-backed artifact にすることである。

## v2.4 Goals

1. role-level execution result を first-class artifact として記録できるようにする
2. team-level aggregation を packet として記録できるようにする
3. council judgment を packet として記録できるようにする
4. execution lineage を task / parent session 単位で inspectable にする
5. `v2.3` operator surface と audit baseline を壊さず、`v2.5` allocation layer の前提を揃える

## Release Gates

### Gate 1: Role Results Are First-Class

- `role-result-record` exists
- artifact is validated against bundled schema
- status / rationale / recommendation / signal list / decision-required が packet に残る

### Gate 2: Team Output Is First-Class

- `team-output-record` exists
- expected / received / missing role state を packet に残せる
- joined role result refs と recommended next step が packet に残る

### Gate 3: Council Review Is First-Class

- `council-review-packet` exists
- review status / decision summary / rationale / recommendation を packet に残せる
- follow-up task ids と escalation flag を packet に残せる

### Gate 4: Execution Lineage Is Inspectable

- `execution-lineage` exists
- role result / team output / council review を集約できる
- source task / parent session / stage filter で slice できる
- active lineage snapshot を `.aof/context/active/execution-lineage.json` に残せる

### Gate 5: Existing Operator Surface Remains Green

- `organization-audit` remains green
- `organization-verify` remains green
- `npm test` remains green
- `npm run smoke` remains green

### Gate 6: Boundary Remains Honest

- `v2.4.0` は governed allocation を claim しない
- `v2.4.0` は policy enforcement runtime を claim しない
- `v2.4.0` は backend-specific orchestration adapter を claim しない
- `v2.5` と `v3.0` の境界を前倒ししない

## Intended Outcome

After `v2.4.0`, AOF operator は current execution flow を次の流れで扱える。

```text
role result
  -> team output
  -> council review
  -> execution lineage snapshot
  -> next task or release judgment
```

この版の価値は、自律実行ではなく、組織の中間判断物を packet contract として固定し、次の governed allocation layer に渡せる状態を作ることにある。
