# AOF v2.3 Release Definition

`v2.3.0` の主題は、`v2.2.0` までに揃った organization / capability baseline を、operator が raw JSON 読みなしで使える runtime surface に進めることである。

## Position

`v2.0.0` は installer と bootstrap を整えた。  
`v2.1.0` は organization / council / team / role / contract / dependency を first-class にした。  
`v2.2.0` は skill / capability / resource / policy / metrics / analytics / learning-loop artifact を追加し、AOF を represent / validate / inspect できる capability baseline にした。  
`v2.3.0` は、その baseline を operator-facing command surface と unified audit path として operationalize する版である。

AOF の本質は依然として organization framework であり、`v2.3.0` でも execution backend や autonomous runtime claim を前倒ししない。  
ここでやるのは、現在の organization state を「見える」「点検できる」「誤魔化せない」形にすることである。

## v2.3 Goals

1. organization state を operator-facing に要約できるようにする
2. contract / dependency / decision / roadmap state を first-class register として返せるようにする
3. unified organization audit surface を提供する
4. task lifecycle integrity を operator-visible にし、重複や衝突を見逃さないようにする
5. `v2.2 = represent / validate / inspect` の境界を保ったまま、current model を operable にする

## Release Gates

### Gate 1: Organization State Is Operator-Visible

- `organization-status` exists
- topology / install mode / write target / goals / councils / teams / roles / task counts を CLI から見られる
- capability-layer presence が要約される

### Gate 2: Contracts And Dependencies Are First-Class Registers

- `contract-register` exists
- `dependency-graph` exists
- contract artifact presence と dependency edge state を raw JSON なしで確認できる

### Gate 3: Decision And Roadmap State Are First-Class Registers

- `decision-register` exists
- `roadmap-status` exists
- decision pair presence と roadmap/backlog mapping を operator-facing に確認できる

### Gate 4: Unified Audit Exists

- `organization-audit` exists
- organization / capability-layer verification summary を束ねる
- decision artifact presence を束ねる
- task lifecycle duplicate detection を含む
- operator-facing audit artifact を残せる

### Gate 5: Runtime Integrity Is Tightened

- concurrent task creation collision risk が軽減されている
- task update で duplicate lifecycle copies が残りにくい
- historical duplicate task ids を audit が検知できる

### Gate 6: Boundary Remains Honest

- `v2.3.0` は execution packet runtime を claim しない
- allocation / enforcement / adaptive runtime は `v2.4+` / `v3.x` に据え置く
- `v2.2` capability baseline を壊さない

## Intended Outcome

After `v2.3.0`, AOF operator は current organization runtime を次の流れで扱える。

```text
Bootstrap / Goals / Organization / Capability Layer
  -> CLI registers and status views
  -> unified audit
  -> trustworthy release planning baseline
```

この版の価値は、新しい runtime autonomy ではなく、既存の organization model を everyday operation に耐える surface へ仕上げることにある。
