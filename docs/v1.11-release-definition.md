# AOF v1.11 Release Definition

AI Organization Framework を `v1.11` として更新・リリースするための基準を定義する。

`v1.11` の主題は、AOF の orchestration layer を **現行 Codex 仕様で実装可能な parent/child execution model** に落とし込むことにある。

## Position

`v1.10` が cadence runtime を human visibility contract と topology-aware GitHub operations policy に接続した版であるなら、  
`v1.11` は **AOF の Orchestrator が現行 Codex 仕様の中でどう child roles を編成し、join し、runtime state に還流するかを fixed model として定義する版** である。

重点は次である。

- Human-started parent Codex orchestration を first-class operating model として明示する
- parent/child role execution contract を fixed shape にする
- child role result artifact と parent join artifact を formalize する
- scheduler-native autonomous Codex startup と、interactive parent orchestration を分離する
- AOF の runtime boundary と Codex orchestration boundary を明示する

## v1.11 Goal

`v1.11` の主目標は次である。

1. AOF の Orchestrator が **現行 Codex 仕様の中で本当にできる loop** を明示すること
2. parent/child role execution を artifact contract と join rule まで含めて formalize すること
3. `Human -> Parent Codex -> Child Roles -> Aggregate -> Runtime Update` を AOF の現実的な orchestration path として定着させること
4. GitHub Actions cadence runtime と Codex Orchestrator を混同しないこと

## In Scope For v1.11

### Parent/Child Orchestration Model

次を fixed model として扱う。

1. Human starts Parent Codex
2. Parent Codex reads and writes `.aof/`
3. Parent Codex spawns role-scoped child agents
4. Child agents return role results
5. Parent Codex joins / aggregates / writes runtime state

### Child Role Result Contract

最低限、child role は次を返せることを定義する。

- `result_type`
- `role`
- `stage`
- `session_id`
- `status`
- `recommendation`
- `rationale`
- `signals`
- `artifact_refs`
- `decision_required`

### Parent Join Contract

最低限、親側の join artifact は次を持てることを定義する。

- `join_type`
- `expected_roles`
- `received_roles`
- `missing_roles`
- `aggregate_state`
- `blocking_signals`
- `recommended_next_step`

### Boundary Clarity

`v1.11` では、少なくとも次の境界を明示する。

- Codex parent orchestration
- Codex child role execution
- deterministic runtime mutation
- external scheduler / cadence runtime

### Current Constraint Formalization

次を **現時点では未実装 / proven architecture ではない** と明示する。

- GitHub Actions -> Codex parent autonomous startup
- runtime -> Codex parent autonomous restart
- unattended continuous Codex loop

## Out Of Scope For v1.11

- full autonomous Codex daemon
- scheduler-triggered Codex startup bridge
- remote hosted orchestration platform
- managed-project bootstrap implementation
- live `.aof` -> visibility producer automation implementation

## Non-Goals

1. `v1.10` cadence visibility / GitHub topology safetyを壊さない
2. 現行 Codex 仕様でできない loop を “できる” と主張しない
3. parent/child parallelism を strong independence claim と混同しない
4. cadence runtime と Codex orchestration を 1 つの層に潰さない

## Release Gates

### Gate 1: v1.10 Integrity Is Preserved

- cadence visibility and topology-aware GitHub operations remain intact
- `v1.11` additions do not weaken the existing self-hosting runtime surfaces

### Gate 2: Parent/Child Orchestration Is Explicit

- parent Codex role is clearly defined
- child role execution pattern is explicit
- loop is described as Human-started interactive orchestration, not autonomous Codex scheduling

### Gate 3: Child Result Contract Is Formalized

- role result artifact shape is documented
- minimum required fields are explicit
- relation between role result and runtime update is explicit

### Gate 4: Join Contract Is Formalized

- parent join artifact shape is documented
- missing role / blocking signal handling is explicit
- aggregation result is expressible in machine-readable form

### Gate 5: Capability Boundary Is Honest

- docs clearly state what current Codex can do
- docs clearly state what still requires a future bridge
- scheduler / cadence automation is not misrepresented as Codex orchestration

### Gate 6: Current AOF State Reflects The New Focus

- next value slice and self-audit reflect the orchestration formalization focus
- at least one current task explicitly tracks the orchestration-model gap

## Deferred To v1.12+

- scheduler-to-parent invocation bridge
- autonomous Codex startup from cadence events
- parent/child execution artifact implementation in runtime commands
- real managed-project examples with Codex orchestration

## Current Intended Shape

`v1.11` は、AOF を

**“cadence runtime and visibility are clearer, but Codex orchestration remains implicit”**

から

**“Codex parent/child orchestration is explicitly modeled, bounded, and honest about current capabilities”**

へ進める版とする。
