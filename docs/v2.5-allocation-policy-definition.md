# AOF v2.5 Allocation And Policy Definition

Date: `2026-06-14`

`v2.5` の最小価値は、AOF が「誰をどこに割り当てるべきか」を artifact として説明できるようにすることである。

## Position

`v2.4` までで AOF は execution packet を持った。  
`v2.5` では、その execution に入る前の governed allocation を first-class にする。

この段階で必要なのは runtime automation ではなく、次の 2 つである。

- allocation recommendation を traceable にする
- policy constraint を execution 前に評価できるようにする
- reviewed recommendation を explicit claim state に変換できるようにする

## Minimal Artifact Set

### Allocation Plan

Purpose:

- role に対して resource をどう割り当てるべきかを説明する

Minimum content:

- subject ref
- target role refs
- candidate resource refs
- recommended allocations
- unfilled role refs
- policy refs
- risk notes

### Policy Evaluation Report

Purpose:

- allocation または execution request に対して、どの policy がどう作用するかを残す

Minimum content:

- subject ref
- evaluation scope
- policy refs
- overall outcome
- per-policy results
- recommended actions

### Resource Claim

Purpose:

- recommendation 済み resource を、approval 状態つきで明示的に確保する

Minimum content:

- subject ref
- resource ref
- claimant role ref
- claim scope
- claim status
- approval policy refs
- justification
- optional allocation plan ref / policy evaluation ref

## Evaluation Rule

`v2.5` で重要なのは、最適解を claim することではない。  
重要なのは、recommendation と constraint を reviewable にすることである。

したがって `v2.5` の allocation layer は次を満たすべきである。

- なぜその resource を候補にしたかが分かる
- どの role がまだ unfilled かが分かる
- どの policy が approval / review / deny を要求したかが分かる
- recommendation が claim requested / approved / denied / released のどこにあるかが分かる
- execution に進む前に council や human が確認できる

## Not Yet Claimed

`v2.5` はまだ次を claim しない。

- automatic staffing engine
- runtime policy enforcement
- autonomous queue-based claiming
- self-directed reassignment
