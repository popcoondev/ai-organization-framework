# AOF v2.2.0 Release Notes

- version: `v2.2.0`
- status: released
- release note revision commit SHA: `bbc2b42`

## Summary

`v2.2.0` is the release that extends AOF from an explicit organization model into an explicit organization capability layer.

This release adds first-class artifacts for:

- skills
- capabilities
- resources
- policy
- metrics snapshots
- organization analytics snapshots
- learning-loop snapshots

It also promotes core benchmark validation work into CLI command surfaces so that organization state can be verified locally and repeatably.

## Highlights

### 1. Capability-Layer Artifacts Are First-Class

`v2.2.0` adds canonical artifact and schema surfaces for:

- `.aof/skills.json`
- `.aof/capability-registry.json`
- `.aof/resource-inventory.json`
- `.aof/policies.json`

These are now part of bootstrap and upgrade flows, rather than optional documentation-only concepts.

### 2. Validation Is Productized

`v2.2.0` adds explicit validation commands for self-hosting and benchmark use:

- `aof organization-verify`
- `aof decision-verify`

`organization-verify` checks:

- organization and capability-layer schema validity
- organization-to-capability-layer reference alignment
- dependency integrity
- contract artifact path existence
- skill / capability / resource / policy cross-reference integrity

`decision-verify` checks:

- decision record schema validity
- filename and `decision_id` alignment
- markdown/json pair completeness
- canonical markdown path integrity

### 3. Metrics, Analytics, And Learning Loop Become Inspectable

`v2.2.0` adds new inspectable runtime artifacts:

- `aof metrics-snapshot`
- `aof organization-analytics-snapshot`
- `aof learning-loop-snapshot`

These commands make organization health, current operating metrics, and improvement signals visible without introducing a separate analytics backend or autonomous optimizer.

### 4. Managed Installation Story Stays Intact

The install/bootstrap path remains simple and tag-based:

```bash
git clone --branch v2.2.0 https://github.com/popcoondev/ai-organization-framework.git ~/.local/share/aof/v2.2.0
cd ~/.local/share/aof/v2.2.0
npm install
npm link
```

Then initialize a managed project with:

```bash
aof init --topology managed-project
```

## Boundary

`v2.2.0` intentionally stops at:

- represent
- validate
- inspect

It does not claim:

- dynamic workforce scheduling
- runtime policy enforcement
- autonomous organization reconfiguration
- adaptive closed-loop optimization

Those remain `v3.x` concerns.
