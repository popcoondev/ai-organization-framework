## Question

What should AOF do next after `v3.5.0` if the goal is to improve real operator effectiveness without weakening runtime truthfulness?

## Proposal Under Review

The reviewed proposal argues that the next problem is not missing runtime capability first, but **visibility composition**:

- status, timeline, and flow are useful but fragmented
- operators can see what is happening more easily than why it happened
- artifact lineage exists in the runtime but is not yet legible from one operator surface
- blockers and recommended next action still require manual reconstruction

The proposal names the target shape `AOF Runtime Mission Control`.

## Validated Need

AOF needs a bounded Mission Control visibility layer that derives mission summary, artifact lineage, blockers, and recommended next action from canonical runtime artifacts.

This is narrower and better than the raw request "turn AOF visibility into Runtime Mission Control" because it identifies:

- the actual bottleneck: mission-level visibility composition
- the truthful product surface: derived summaries and causal graph views
- the non-goals: ornamental UI, speculative liveness, and duplicate truth layers

## Why This Is A Good Next Goal

- It builds directly on `v3.5.0`.
- It solves the next observed operator bottleneck after command routing improved.
- It can be implemented as additive visibility composition rather than a new authority layer.
- It preserves Discovery / Need Validation / execution boundaries.
- It keeps the runtime honest by requiring Mission Control to derive from canonical artifacts.

## Recommended Release Shape

Proceed with:

- `v3.6.0 = bounded Mission Control visibility layer`

Core release claims should be limited to:

- mission overview
- artifact graph or equivalent causal chain view
- blocker visibility
- recommended next action visibility
- verification that these surfaces remain derived from canonical runtime state

## Deferred

Do not include these in `v3.6.0`:

- pixel-office presentation
- richer live role-state or council-state claims without truthful derivation
- broader autonomy claims hidden inside visibility work
- any second source of truth outside runtime artifacts
