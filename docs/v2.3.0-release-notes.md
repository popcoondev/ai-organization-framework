# AOF v2.3.0 Release Notes

- version: `v2.3.0`
- status: draft
- release note revision commit SHA: `TBD (uncommitted release candidate worktree)`

## Summary

`v2.3.0` is the release that makes the current AOF organization and capability model operator-usable without requiring direct JSON inspection.

This release adds first-class operator surfaces for:

- organization status
- contract register
- dependency graph
- decision register
- roadmap status
- unified organization audit

It also tightens runtime task integrity so that the new operator surface is trustworthy rather than merely readable.

## Highlights

### 1. The Organization Model Becomes Operable

`v2.3.0` adds new CLI surfaces for reading current AOF state as a runtime, not only as files:

- `aof organization-status`
- `aof contract-register`
- `aof dependency-graph`
- `aof decision-register`
- `aof roadmap-status`

These commands expose topology, goals, contracts, dependencies, decisions, and release-track task groupings through operator-facing output.

### 2. Unified Audit Is Now First-Class

`v2.3.0` adds:

- `aof organization-audit`

This command bundles:

- organization and capability-layer verification summary
- decision artifact presence summary
- task lifecycle duplicate detection
- audit artifact writeback to `.aof/context/active/organization-audit.json`

This gives operators one integrity entrypoint before moving on to deeper verification or release decisions.

### 3. Runtime Integrity Was Hardened

To make operator surfaces credible, `v2.3.0` also improves the underlying task-memory behavior:

- project-memory task mutations now use serialized locking
- task updates consolidate duplicate lifecycle copies
- historical duplicate task ids are surfaced explicitly by audit

The self-hosting repo used these same runtime surfaces to detect and reconcile a real historical task collision before claiming release readiness.

### 4. The v2.2 Boundary Still Holds

`v2.3.0` does not claim:

- execution packet runtime
- governed allocation
- policy enforcement
- autonomous orchestration

Those remain later scope for `v2.4+` and `v3.x`.

## Install / Upgrade

Current canonical acquisition path:

```bash
git clone --branch v2.3.0 https://github.com/popcoondev/ai-organization-framework.git ~/.local/share/aof/v2.3.0
cd ~/.local/share/aof/v2.3.0
npm install
npm link
```

Then initialize another repository with:

```bash
aof init --topology managed-project
```

Or migrate an existing bootstrap shape with:

```bash
aof upgrade
```

## Boundary

`v2.3.0` is the release where AOF becomes operable at the operator surface.

It still stops at:

- represent
- validate
- inspect

The next bridge release, `v2.4.0`, is where execution packet and judgment artifacts should begin.
