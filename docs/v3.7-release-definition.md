# AOF v3.7 Release Definition

`v3.7.0` の主題は、
**runtime situation assessment と roadmap truthfulness を release-grade にすること**
である。

## Position

`v3.6.0` は bounded Mission Control visibility を成立させた。  
しかし release 後に runtime を実際に回すと、次の friction が残った。

- active release は正しくても current guidance が前 release の task を引きずりうる
- `roadmap-status` が historical alignment を current truth より強く扱いうる
- Mission Control が current mission を見せても、いま何が矛盾しているかは operator が判断し直す必要がある
- shipped work が open task として残ると runtime 自体が stale recommendation を返しうる

したがって `v3.7.0` でやるべきことは viewer 拡張ではない。  
やるべきことは **runtime 自身が current truth conflict を診断し、正しい frontier を指せるようにすること** である。

## v3.7 Goals

1. current release baseline と open frontier の矛盾を runtime が診断できる
2. `roadmap-status` が active release truth と live frontier に揃う
3. Mission Control が stale shipped work ではなく current frontier を推奨する
4. release 後の self-hosting state が bounded repair path で refresh できる
5. situation judgment が dedicated command と test で検証できる

## Out Of Scope

- decorative viewer work beyond bounded truth surfaces
- speculative live role-state or council-state claims
- Discovery / Need Validation authority changes
- broader organization analytics expansion
- autonomy expansion driven by stale state
- any handwritten second source of truth outside canonical artifacts

## Release Gates

### Gate 1: Situation Diagnosis Is First-Class

- a dedicated runtime command emits a situation assessment packet
- the packet names current runtime stage, active release, primary frontier, conflicts, and recommended action

### Gate 2: Release Completion Stops Being Current Work

- shipped release work no longer remains the current recommended task
- release-complete implementation tasks can be reconciled through one bounded runtime path

### Gate 3: Roadmap Guidance Matches Active Truth

- `roadmap-status` derives current alignment from active release truth and current frontier
- historical decomposition does not override present operator guidance

### Gate 4: Mission Control Uses Situation Judgment

- Mission Control surfaces current truth conflicts from runtime-derived diagnosis
- Mission Control recommended next action follows the same frontier as situation assessment

### Gate 5: Boundary Remains Honest

- situation assessment does not invent new execution authority
- Discovery remains evidence
- Need Validation remains approval authority
- diagnosis remains artifact-derived and backend-neutral

## Intended Outcome

After `v3.7.0`, the runtime should be able to say, and verify, one coherent thing:

```text
this is the current release baseline
this is the real current frontier
this is where truth conflicts exist
this is the next operating move
and those judgments are derived from canonical artifacts rather than a separate operator story
```
