# AOF v2.6 Release Definition

`v2.6.0` の主題は、Human Visibility Layer を live `.aof` state から自動生成できる runtime-backed visibility projection に進めることである。

## Position

`v2.5.0` は governed allocation を first-class artifact にした。  
`v2.6.0` は、その runtime state を operator-facing visibility packet に自動投影し、手動 fixture 依存を外す版である。

ここでやるのは hosted dashboard service ではない。  
`status_card`、`timeline_feed`、`flow_snapshot` を current runtime artifact から生成し、既存 `visibility-serve` に接続することである。

## v2.6 Goals

1. runtime-backed `status_card` を first-class に出力できるようにする
2. runtime-backed `timeline_feed` を first-class に出力できるようにする
3. runtime-backed `flow_snapshot` を first-class に出力できるようにする
4. generated visibility views が既存 `visibility-serve` viewer にそのまま接続できるようにする
5. `v2.5` allocation / metrics / audit surfaces を壊さず、`v3.0` runtime proof の operator visibility 基盤を揃える

## Release Gates

### Gate 1: Status Card Is Runtime-Backed

- `visibility-export` exists
- generated `status_card` is schema-backed
- current phase, current goal, next checkpoint, and evidence state are exported from live runtime state

### Gate 2: Timeline Feed Is Runtime-Backed

- generated `timeline_feed` is schema-backed
- entries are derived from current runtime artifacts rather than fixture-only input
- timeline entries expose refs back to canonical `.aof` artifacts

### Gate 3: Flow Snapshot Is Runtime-Backed

- generated `flow_snapshot` is schema-backed
- ordered nodes expose current bridge-release progress
- `v2.6` visibility work is represented in the operator-facing flow while open and transitions to `done` when the bridge release closes

### Gate 4: Viewer Integration Remains Coherent

- generated `status_card`, `timeline_feed`, and `flow_snapshot` are consumable by `visibility-serve`
- `roadmap-status` exposes `v2.6` as a first-class release track
- `roadmap-status` points to `docs/v2.6-release-definition.md`

### Gate 5: Existing Runtime Baseline Remains Green

- `organization-audit` remains green
- `organization-verify` remains green
- `npm test` remains green
- `npm run smoke` remains green

### Gate 6: Boundary Remains Honest

- `v2.6.0` does not claim hosted visibility infrastructure
- `v2.6.0` does not claim backend-neutral execution proof
- `v2.6.0` does not claim autonomous orchestration
- `v3.0` remains the first release for auditable end-to-end runtime proof

## Intended Outcome

After `v2.6.0`, AOF operator は current runtime state を次の流れで扱える。

```text
live .aof artifacts
  -> status_card
  -> timeline_feed
  -> flow_snapshot
  -> visibility-serve viewer
  -> operator-facing release judgment
```

この版の価値は、新しい orchestration claim ではなく、既存 runtime state を operator-facing visibility layer に自動投影し、`v3.0` proof の観測面を固めることにある。
